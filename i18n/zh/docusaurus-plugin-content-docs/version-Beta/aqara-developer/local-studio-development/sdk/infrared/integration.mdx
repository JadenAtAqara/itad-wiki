# 集成红外遥控控制 SDK（含 UI）

## 概述

Studio 红外遥控（带 UI）SDK 是基于 React Native 插件 `rn_device_ir` 提供的完整红外遥控能力，面向 **Aqara Studio** 场景。宿主 App 通过 Lumi React Native SDK 加载插件 Bundle，并传入 Studio OpenAPI 鉴权信息与目标页面，即可在 App 内打开：

- **红外设备创建流程**（选网关 → 命名 → 品类匹配 → 码库/学习 → 保存）
- **红外设备控制页面**（按设备类型展示对应遥控 UI）
- **Studio 调试入口**（仅开发联调使用，含登录与 OpenAPI 调试能力）

SDK 内置 Aqara UX 设计的完整红外交互页面，宿主无需自行开发。

### RN Bundle 加载方式

示例工程使用 **预制 Bundle** 模式，Bundle 随 `reactnative` SDK 一并下发，无需手动拷贝 JS 包：

```kotlin
LumiReactNativeManager.getInstance().init(
    application,
    LumiReactNativeManager.LumiReactNativeConfig.builder()
        .viewRenderType(RNViewRenderType.STANDARD)
        .bundleLoadType(RNBundleLoadType.PREFAB_ASSETS)
        .bundleConfigLoadType(RNBundleConfigLoadType.PREFAB_ASSETS)
        .build()
)
```

插件标识固定为 **`rn_device_ir`**。若使用自定义 Bundle 分发策略，请与 SDK 团队确认 Bundle 放置路径与版本匹配规则。

### 本地预载 RN Bundle

请将示例工程 `assets` 目录下的 `lumi` 文件夹拷贝至宿主的 `assets` 目录中。

## SDK 初始化

在打开任何 RN 页面前，必须先完成 Core 与 RN 初始化。参考 `MainActivity.init()`：

```kotlin
private fun init() {
    // 1. 初始化 Lumi Core
    LumiCoreManager.getInstance().init(
        application, true, LumiCoreManager
            .LumiCoreSDKConfig.builder()
            .sdkChannel("Aqara Studio")
            .build()
    )

    // 2. 初始化 RN 容器
    LumiReactNativeManager.getInstance().init(
        application,
        LumiReactNativeManager.LumiReactNativeConfig.builder()
            .viewRenderType(RNViewRenderType.STANDARD)
            .bundleLoadType(RNBundleLoadType.PREFAB_ASSETS)
            .bundleConfigLoadType(RNBundleConfigLoadType.PREFAB_ASSETS)
            .build()
    )

    // 3.（可选）注册 RN → Native 消息回调
    LumiReactNativeManager.getInstance().registerRNMessageCallback(object : IRNMessageCallback {
        override fun onHandleMessage(jsonMessage: String, promise: Promise?) {
            // 处理插件回传消息
        }
    })
}
```

## 启动 RN 页面

统一通过 `LumiReactNativeManager.startRNPage` 打开插件页面：

```kotlin
LumiReactNativeManager.getInstance().startRNPage(
    pluginName,   // 固定："rn_device_ir"
    paramsJson,   // 启动参数 JSON 字符串
    entrancePage  // 入口页面名称
)
```

| 参数 | 类型 | 说明 |
| --- | --- | --- |
| `pluginName` | `String` | RN 插件名，固定传 `"rn_device_ir"` |
| `paramsJson` | `String` | 页面启动参数的 JSON 字符串 |
| `entrancePage` | `String` | 入口 Route 名称，见 [支持的入口页面](#支持的入口页面) |

## 支持的入口页面

| 入口页面 | 用途 | 推荐场景 |
| --- | --- | --- |
| `LoginStudioPage` | Studio 账号密码登录 + 调试首页 | **仅开发联调** |
| `CreateIrDevicePage` | 红外虚拟设备创建（带完整 UI 流程） | **生产接入** |
| `ControlIrDevicePage` | 直接进入红外遥控控制页 | **生产接入** |

插件内部会根据 `entrancePage` 决定初始 Route（见 `Entry.tsx` 中 `getInitialRouteName`）。

## 鉴权信息获取

生产接入时，宿主 App 需自行完成 Studio 用户鉴权，并将以下信息传入插件：

- `studioHost`
- `studioAccessToken`
- `studioRefreshToken`
- `studioClientId`

获取方式取决于 Studio 部署的 OAuth / 登录方案，通常由 Studio 服务端或配套认证接口颁发。示例工程中的 Token 仅用于本地联调，**请勿在生产环境硬编码**。

调试阶段也可使用 `LoginStudioPage` 在插件内完成账号密码登录，登录成功后插件会写入 OpenAPI Session，可在调试首页继续验证创建设备等能力。

## 消息回调

通过 `registerRNMessageCallback` 可接收插件向 Native 回传的 JSON 消息，便于宿主统一处理退出、埋点等事件：

```kotlin
LumiReactNativeManager.getInstance().registerRNMessageCallback(object : IRNMessageCallback {
    override fun onHandleMessage(jsonMessage: String, promise: Promise?) {
        Logs.e("RN Message: $jsonMessage")
    }
})
```
