# JavaScript 脚本（适用于 MQTT）

本文件介绍了如何通过 JavaScript 脚本在 Aqara Studio 中自定义添加 MQTT 设备功能点及其解析方法。

## 脚本介绍

本脚本**必须**包含以下 4 个 JavaScript 方法：
- [parseDiscoveryResponse](#parsediscoveryresponse)
- [parseDiscoverPointsResponse](#parsediscoverpointsresponse)
- [parseReportRequest](#parsereportrequest)
- [buildWriteRequest](#buildwriterequest)

:::tip
请勿更改方法的参数及其输入输出结构。
:::

## 控制台日志

脚本支持使用 `console.log`、`console.info`、`console.debug`、`console.warn` 和 `console.error` 输出调试日志。多个参数会以空格拼接，`warn` 和 `error` 日志分别带有 `[warn]` 和 `[error]` 前缀。日志显示在协议配置页的只读字段 `Console Content` 中，最多保留 2048 个字符；超出限制时旧内容会被替换。每个脚本解析器每秒最多接收 20 条日志，超出部分会被忽略并在下一时间窗口输出汇总提示；单条日志最多保留 512 个字符并转换前 16 个参数。请勿在循环或高频报文处理函数中无条件输出日志，调试时应使用条件开关或采样输出。

## parseDiscoveryResponse

该方法 `parseDiscoveryResponse` 主要用于向 Aqara Studio 声明需要接入的 MQTT 设备的基本配置信息。

您需要以**对象数组**的形式返回待接入设备的基本信息，详情请参考 [设备对象结构](#设备对象结构)。

Aqara Studio 会调用此方法以发现所有可用设备，实现自动接入与管理。

### 设备对象结构

| 字段名称 | 说明 |
| --- | --- |
| deviceName | 设备名称。 |
| deviceId   | 设备 ID。 |
| brokerEndpoint | MQTT Broker 的地址。 |
| brokerPort | MQTT Broker 的端口。 |
| connectionType | 连接类型。支持以下取值：<ul><li>0：匿名</li><li>1：通过 SSL 匿名</li><li>2：通过 SSL 用户登录</li><li>3：用户登录无 SSL</li></ul> |
| username | 用户名。 |
| password | 密码。 |

字段必填说明：
- `deviceId`、`brokerEndpoint`、`brokerPort`、`connectionType` 必填。
- `brokerPort` 默认端口为 `1883`，`connectionType` 默认值为 `3`。
- `username`/`password` 按 `connectionType` 决定是否需要。
- `deviceName` 可选，为空时使用空字符串。

### 示例 

```javascript
function parseDiscoveryResponse() {
	return [
		{
			deviceName: "TestDevice",
			deviceId: "98CC4D189E60",
			brokerEndpoint: "127.0.0.1",
			brokerPort: 1883,
			connectionType: 3,
			username: "admin",
			password: "public"
		}
	];
}
```

## parseDiscoverPointsResponse

该方法负责向 Aqara Studio 声明“该设备具备哪些功能点”。Aqara Studio 调用该函数时会把 `deviceId` 作为第一个参数传入。

您需要在返回数组中填写一个或多个待发现功能点结构（如 `pointId` 等，详情请见 [point 对象结构](#point-对象结构)）。这样，Aqara Studio 才能在功能点发现流程中会调用该函数，解析返回值，从而识别设备具备的所有功能点，实现功能点无缝接入和控制。

### point 对象结构

| 字段名称    | 说明                                                                                     |
|-----------|-------------------------------------------------------------------------------------------|
| pointId   | 功能点 ID。请咨询设备厂商获取功能点的真实 ID，以便 Aqara Studio 发现该点。                                                    |
| pointName | 您自定义的功能点名称。                                                                              |
| devType | 对应 Aqara 物模型规范的 `deviceType`。具体可用值请阅读 [设备类型](../aqara-developer/aqara-data-model-specification/device-types.mdx)。                                  |
| functionCode | 对应 Aqara 物模型规范的 `functionCode`，具体可用值请阅读 [功能代码](../aqara-developer/aqara-data-model-specification/function-codes.mdx)。                                  |
| traitCode   | 对应 Aqara 物模型规范的 `traitCode`，具体可用值请阅读 [功能点代码](../aqara-developer/aqara-data-model-specification/trait-codes.mdx)。                                      |
| dataType  | 功能点数据类型，支持以下取值：<ul><li><code>bool</code>：布尔型</li><li><code>numerical</code>：数值型</li><li><code>enum</code>：枚举型</li><li><code>text</code>：文本型</li></ul> |
| enumRange | 枚举选项数组。仅当通用功能点的 `dataType` 为 `enum` 时必填。数组不能为空；每项包含整数类型的 `value` 和字符串类型的 `label`，`value` 和 `label` 均不可重复，且 `label` 不可为空。 |
| min | 数值型通用功能点的最小值。仅当 `dataType` 为 `numerical` 时生效；也可使用 `minimum` 作为别名。 |
| max | 数值型通用功能点的最大值。仅当 `dataType` 为 `numerical` 时生效；也可使用 `maximum` 作为别名。 |
| step | 数值型通用功能点的步长，必须大于 0。仅当 `dataType` 为 `numerical` 时生效；也可使用 `resolution` 作为别名。 |
| precision | 数值型通用功能点的精度值，必须大于等于 0。仅当 `dataType` 为 `numerical` 时生效。 |
| decimals | 数值型通用功能点的小数位数，必须为大于等于 0 的整数。仅当 `dataType` 为 `numerical` 时生效。 |
| access    | 读写能力标识，<ul><li>1：可读</li><li>2：可写</li><li>3：可读可写</li></ul>                                 |
| subTopic  | （**仅适用于 MQTT 设备**）Aqara Studio 通过订阅该 Topic 获取功能点的最新数据。具体 Topic 请根据设备协议或厂商文档填写；需要同时接收多个上行主题时，可按 MQTT 规范填写 `+` 或 `#` 通配符。  |
| pubTopic  | （**仅适用于 MQTT 设备**）Aqara Studio 通过向此 Topic 发送消息数据控制功能点。具体 Topic 请根据设备协议或厂商文档填写。（对于仅可读的功能点，此字段留空）        |

:::note
对于 `numerical` 点，Aqara Studio 在写值调用 `buildWriteRequest` 时会优先依据 `decimals` 格式化数值；未配置 `decimals` 时会依据 `step` / `resolution` 推导小数位。为了避免 `0.3` 被二进制浮点误差表现为 `0.30000030249357224`，建议需要小数写入的点显式配置 `decimals` 或 `step`。`precision` 仅用于描述精度，不作为写值报文的小数位截断规则。
:::

:::tip
您可以根据实际情况选择以下任意方式构造功能点的数据结构：   
- **基于 Aqara Spec**：需填写 `devType`、`functionCode`、`traitCode` 等字段，这可以将设备的功能点关联到 Aqara 物模型规范协议中的对应功能点。
- **通用方式**：需填写 `dataType`、`access` 等基本字段。
:::

## parseReportRequest

该方法负责从设备上报的数据解析出指定功能点（`pointId`）的当前值。

当 Aqara Studio 收到设备上报消息时，会调用此函数并传入功能点 ID 和原始消息字符串。您需要在本函数中实现从其中的 `values` 字段提取出对应 `pointId` 的值。这样 Aqara Studio 才能实时更新与展示功能点值。

:::tip
请结合设备的实际数据上报协议，自定义解析逻辑，确保能准确从原始消息中提取 `pointId` 对应的数值（例如示例中的 `obj["values"][pointId]`）。
:::

### 示例

```javascript
// 以下为解析 MQTT 设备上报 JSON 消息字符串的示例
function parseReportRequest(pointId, jsonStr) {
	try {
		const obj = JSON.parse(jsonStr);
		return obj["values"][pointId]?.toString();
	} catch (e) {
		return null;
	}
}
```

## buildWriteRequest

该方法负责根据要控制的功能点 ID 和设定的值，构造向设备下发控制指令的字符串。

当 Aqara Studio 需要向设备发送控制命令（如开关、设定温度）时，会调用此函数，传入设备 ID、目标功能点 ID 和要设置的数值。函数需返回一个字符串，该字符串需符合设备协议的控制指令格式，通常包含设备标识、时间戳和具体的值字段。

请根据设备实际可接受的报文格式，在函数内部合理构建对象结构。例如，以 MQTT 设备为例，可包含 `node`、`group`、`timestamp` 等字段，并将 `pointId` 对应的值被写入 `values` 字段。

### 示例

```javascript
// 以通过 JSON 字符串控制 MQTT 设备为例
// value 参数由 Aqara Studio 自动传入，无需手动声明或赋值
function buildWriteRequest(deviceId, pointId, value) {
	try {
		const obj = {
			"node": "冷机",
			"group": "1#冷机",
			"timestamp": 1768204054654,
			"values": {
				
			},
			"errors": {
				
			},
			"metas": {
				
			}
		};
		obj.values[pointId] = parseFloat(value);
		return JSON.stringify(obj);
	} catch (e) {
		return null;
	}
}
```

## 完整示例

下面是一个添加 MQTT 设备功能点的完整示例：

示例中的 `globalThis` 注册是动态的：脚本实际实现了哪些函数，就只注册哪些函数；未实现的函数不需要注册。

:::note
曼顿 MQTT 协议中，`/status/json` 是周期性状态上报，`/event/json` 是状态改变时上报，`/resp/json` 是设备对 `/cmd/json` 的指令回复。若希望控制后更快刷新点位，可以将点位订阅配置为 `/dev/MTS/{DEVID}/+/json`，并在 `parseReportRequest` 中同时识别 `status`、`event` 和 `resp` 报文；`resp` 只能代表命令回复，最终状态仍以 `event/status` 上报为准。
:::

```javascript
(function() {
	var DEVICE_ID = "98CC4D189E60";
	var CATE = "MTS";
	var SUB_TOPIC = "/dev/" + CATE + "/" + DEVICE_ID + "/+/json";
	var PUB_TOPIC = "/dev/" + CATE + "/" + DEVICE_ID + "/cmd/json";
	var GATEWAY_DST_ID = 43008827855272960;

	var nextPackIdValue = 180289391;
	var pendingWrites = {};
	var pendingPackIds = [];

	function parseDiscoveryResponse() {
		return [
			{
				deviceName: "曼顿空开",
				deviceId: DEVICE_ID,
				brokerEndpoint: "127.0.0.1",
				brokerPort: 1883,
				connectionType: 3,
				username: "admin",
				password: "public"
			}
		];
	}

	function parseDiscoverPointsResponse(deviceId) {
		return [
			{
				pointId: "busAddr1.on",
				pointName: "开关1",
				dataType: "bool",
				access: 3,
				subTopic: SUB_TOPIC,
				pubTopic: PUB_TOPIC
			},
			{
				pointId: "busAddr2.on",
				pointName: "开关2",
				dataType: "bool",
				access: 3,
				subTopic: SUB_TOPIC,
				pubTopic: PUB_TOPIC
			}
		];
	}

	function parseReportRequest(pointId, jsonStr) {
		try {
			var busAddr = getPointBusAddr(pointId);
			if (busAddr === null) {
				return null;
			}

			var obj = JSON.parse(jsonStr);
			var statusValue = parseStatusValue(busAddr, obj);
			if (statusValue !== null) {
				return statusValue;
			}

			var eventValue = parseEventValue(busAddr, obj, jsonStr);
			if (eventValue !== null) {
				return eventValue;
			}

			return parseCommandResponseValue(busAddr, obj);
		} catch (e) {
			return null;
		}
	}

	function parseStatusValue(busAddr, obj) {
		var data = obj && Array.isArray(obj.data) ? obj.data : [];
		for (var i = 0; i < data.length; i++) {
			var item = data[i] || {};
			var state = item.mcbComSta || item["mcbComSta "];
			if (state && sameBusAddr(state.busAddr, busAddr) && state.on !== undefined && state.on !== null) {
				return normalizeOnValue(state.on);
			}

			if (sameBusAddr(item.busAddr, busAddr) && item.on !== undefined && item.on !== null) {
				return normalizeOnValue(item.on);
			}
		}
		return null;
	}

	function parseEventValue(busAddr, obj, jsonStr) {
		var data = obj && Array.isArray(obj.data) ? obj.data : [];
		var rawDevIds = extractRawDevIds(jsonStr);

		for (var i = 0; i < data.length; i++) {
			var item = data[i] || {};
			if (!isEventItem(item)) {
				continue;
			}

			var eventBusAddr = resolveEventBusAddr(item, rawDevIds[i]);
			if (eventBusAddr === null || eventBusAddr !== busAddr) {
				continue;
			}

			var value = eventToOnValue(item);
			if (value !== null) {
				return value;
			}
		}
		return null;
	}

	function parseCommandResponseValue(busAddr, obj) {
		if (!obj || Number(obj.method) !== 13 || Number(obj.msgCode) !== 120 || obj.packID === undefined) {
			return null;
		}

		var key = String(obj.packID);
		var pending = pendingWrites[key];
		if (!pending || Number(pending.busAddr) !== busAddr) {
			return null;
		}

		delete pendingWrites[key];
		if (Number(obj.retCode) >= 200 && Number(obj.retCode) < 300) {
			return pending.value;
		}
		return null;
	}

	function buildWriteRequest(deviceId, pointId, value) {
		try {
			var busAddr = getPointBusAddr(pointId);
			if (busAddr === null) {
				return null;
			}

			var isOn = isOnValue(value);
			var packID = nextPackId();
			rememberPendingWrite(packID, busAddr, isOn ? "1" : "0");

			var obj = {
				method: 3,
				packID: packID,
				dstID: GATEWAY_DST_ID,
				msgCode: 120,
				timeMS: 0,
				priority: 2,
				data: [
					{
						tag: 305,
						cmd: isOn ? "do_turn_on" : "do_turn_off",
						count: 1,
						busAddr: [busAddr]
					}
				]
			};
			return JSON.stringify(obj);
		} catch (e) {
			return null;
		}
	}

	function getPointBusAddr(pointId) {
		var match = /^busAddr(\d+)\.on$/.exec(pointId || "");
		if (!match) {
			return null;
		}
		var busAddr = parseInt(match[1], 10);
		return isValidBusAddr(busAddr) ? busAddr : null;
	}

	function sameBusAddr(value, busAddr) {
		return Number(value) === busAddr;
	}

	function isValidBusAddr(value) {
		return Number(value) >= 1 && Number(value) <= 120;
	}

	function isOnValue(value) {
		if (value === true || value === 1) {
			return true;
		}
		var text = String(value).toLowerCase();
		return text === "1" || text === "true" || text === "on";
	}

	function normalizeOnValue(value) {
		if (value === true) {
			return "1";
		}
		if (value === false) {
			return "0";
		}
		var text = String(value).toLowerCase();
		if (text === "true" || text === "on" || text === "1") {
			return "1";
		}
		if (text === "false" || text === "off" || text === "0") {
			return "0";
		}
		var numberValue = Number(value);
		if (isNaN(numberValue)) {
			return null;
		}
		return numberValue === 0 ? "0" : "1";
	}

	function isEventItem(item) {
		return item && (
			item.eCode !== undefined ||
			item.eInfo !== undefined ||
			item.eSN !== undefined ||
			item.eResult !== undefined
		);
	}

	function eventToOnValue(item) {
		var code = Number(item.eCode !== undefined ? item.eCode : item.code);
		if (code === 33) {
			return "0";
		}
		if (code === 34) {
			return "1";
		}

		var info = String(item.eInfo || item.info || "");
		if (info.indexOf("分闸") >= 0) {
			return "0";
		}
		if (info.indexOf("合闸") >= 0) {
			return "1";
		}
		return null;
	}

	function resolveEventBusAddr(item, rawDevId) {
		var fields = ["busAddr", "addr", "devAddr"];
		for (var i = 0; i < fields.length; i++) {
			var value = item[fields[i]];
			if (value !== undefined && value !== null && isValidBusAddr(Number(value))) {
				return Number(value);
			}
		}
		return busAddrFromDevId(item.devID, rawDevId);
	}

	function busAddrFromDevId(value, rawValue) {
		var text = rawValue !== undefined && rawValue !== null ? String(rawValue) : String(value);
		if (!/^\d+$/.test(text)) {
			return null;
		}
		if (text.length <= 3) {
			var smallValue = Number(text);
			return isValidBusAddr(smallValue) ? smallValue : null;
		}
		var lowByte = decimalStringMod(text, 256);
		return isValidBusAddr(lowByte) ? lowByte : null;
	}

	function decimalStringMod(text, mod) {
		var result = 0;
		for (var i = 0; i < text.length; i++) {
			result = (result * 10 + (text.charCodeAt(i) - 48)) % mod;
		}
		return result;
	}

	function extractRawDevIds(jsonStr) {
		var result = [];
		var regex = /"devID"\s*:\s*"?(\d+)"?/g;
		var match;
		while ((match = regex.exec(jsonStr)) !== null) {
			result.push(match[1]);
		}
		return result;
	}

	function nextPackId() {
		nextPackIdValue += 1;
		if (nextPackIdValue > 999999999) {
			nextPackIdValue = 180289392;
		}
		return nextPackIdValue;
	}

	function rememberPendingWrite(packID, busAddr, value) {
		var key = String(packID);
		pendingWrites[key] = {
			busAddr: busAddr,
			value: value
		};
		pendingPackIds.push(key);
		while (pendingPackIds.length > 32) {
			delete pendingWrites[pendingPackIds.shift()];
		}
	}

	globalThis.parseDiscoveryResponse = parseDiscoveryResponse;
	globalThis.parseDiscoverPointsResponse = parseDiscoverPointsResponse;
	globalThis.buildWriteRequest = buildWriteRequest;
	globalThis.parseReportRequest = parseReportRequest;
	return null;
})();
```
