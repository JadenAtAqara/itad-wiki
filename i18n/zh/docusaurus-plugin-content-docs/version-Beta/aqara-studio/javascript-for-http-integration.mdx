# JavaScript 脚本（适用于 HTTP）

本文件介绍了如何通过 JavaScript 脚本在 Aqara Studio 中自定义添加 HTTP 设备和功能点及其解析方法。

## 脚本介绍

您在本脚本需提供发起设备 HTTP 请求所需的信息，并自定义如何将返回数据解析为 Aqara Studio 能识别的标准格式。

本脚本**必须**包含以下 9 个 JavaScript 方法：

| 方法 | 说明 |
| --- | --- |
| [buildDiscoveryRequest](#builddiscoveryrequest) | 用于生成发现设备的请求报文 |
| [parseDiscoveryResponse](#parsediscoveryresponse) | 用于解析发现设备的响应报文 |
| [buildDiscoverPointsRequest](#builddiscoverpointsrequest) | 用于生成查询设备点（属性/功能点）信息的请求报文 |
| [parseDiscoverPointsResponse](#parsediscoverpointsresponse) | 用于解析设备点信息的响应报文，返回点的详细信息 |
| [buildReadRequest](#buildreadrequest) | 用于生成读取设备点数据的请求报文 |
| [parseReadResponse](#parsereadresponse) | 用于解析读取点数据的响应报文，返回点的值 |
| [buildWriteRequest](#buildwriterequest) | 用于生成写入设备点数据的请求报文 |
| [parseWriteResponse](#parsewriteresponse) | 用于解析写入点数据的响应报文，返回是否成功 |
| [parseReportRequest](#parsereportrequest) | 用于解析设备上报的报文，提取设备ID、点ID、点值等信息 |

:::tip
请勿更改方法的名称和参数。
:::

## 前提条件

请确保您已掌握待接入设备的 HTTP 接口信息，包括请求地址、请求方法、请求头及请求体参数等。

## buildDiscoveryRequest

请在该方法中返回一个请求配置对象，用于指示 Aqara Studio 如何通过 HTTP API 发现设备。

### 入参说明

本函数应当无入参，否则 Aqara Studio 无法正确发起请求。

### 返回值

为确保 Aqara Studio 可以正常向设备发起请求，您需要确保 buildDiscoveryRequest 函数返回的数据的格式与下列一致：

```json
{
    "path": "/report/discoveryDevices",
    "method": "POST",
    "headers": "{\"Content-Type\":\"application/json\"}",
    "postParams": "{\"appId\":\"V0001\"}"
}
```

字段说明如下：

| 字段         | 类型     | 含义 |
|--------------|----------|------|
| path         | String   | 请求路径。用于指定设备发现接口地址，例如 `/report/discoveryDevices`。<br />如果 `method` 为 `GET` 且包含参数，可直接拼接到 path，例如 `/discoveryDevices?channel=1`。 |
| method       | String   | HTTP 请求方法，支持 `GET` 和 `POST`，用于指定调用发现接口时使用的 HTTP 请求方式。 |
| headers      | String   | 请求头，**JSON 字符串格式**（例如：`{"Content-Type":"application/json"}`） |
| postParams   | String   | POST 请求体参数，**JSON 字符串格式**（例如：`{"appId":"V0001"}`）。 |

:::tip
请根据设备官方技术文档或联系设备技术支持获取 `path`、`method`、`headers` 及 `postParams`。
:::

### 函数示例
    
```javascript
function buildDiscoveryRequest() {
    return {
        path: "/report/discoveryDevices",
        method: "POST",
        headers: JSON.stringify({
            "Content-Type": "application/json"
        }),
        postParams: JSON.stringify({
            "appId": "V0001"
        })
    };
}
```

## parseDiscoveryResponse  

请在此方法中解析设备发现接口（[buildDiscoveryRequest](#builddiscoveryrequest)）的返回结果，并返回包含设备 ID（deviceId）和设备名称（deviceName）的设备列表对象数组，供 Aqara Studio 使用。

### 入参说明

本函数的入参应仅有一个，且参数名必须为 responseString，否则 Aqara Studio 无法正确传递和解析数据。

### 返回值

为确保 Aqara Studio 可以正常获取设备信息，您需要确保 parseDiscoveryResponse 函数返回的数据的格式与下列一致：

```json
[
    {
        "deviceId": "123456",
        "deviceName": "温度传感器"
    }
]
```

### 函数示例

假设设备发现接口返回的数据如下：

```json
{
    "reply": {
        "data": {
            "meta": {
                "limit": 10,
                "offset": 0
            },
            "list": [
                {
                    "deviceId": "123456",
                    "deviceName": "test1",
                    "ip": "127.0.0.1",
                    "createTime": "qwerasd"
                }
            ]
        },
        "returnCode": {
            "type": "S",
            "code": "AAAAA",
            "domain": null
        }
    }
}
```

如需处理上述数据，您可参考以下示例设计此函数：

```javascript
function parseDiscoveryResponse(responseString) {
    if (!responseString) return [];
    let obj;
    try {
        obj = JSON.parse(responseString);
    } catch (e) {
        return [];
    }
    const list = obj?.reply?.data?.list;
    if (!Array.isArray(list)) return [];
    // 返回数组，每个元素包含 deviceId 和 deviceName
    return list.map(item => ({
        deviceId: item.deviceId,
        deviceName: item.deviceName
    }));
}
```

## buildDiscoverPointsRequest  

请在该方法中返回一个请求配置对象，用于指示 Aqara Studio 如何通过 HTTP API 查询指定设备的功能点（points）。

### 入参说明

本函数应当无入参，否则 Aqara Studio 无法正确发起请求。

### 返回值

为确保 Aqara Studio 可以正常向设备发起请求，您需要确保 buildDiscoveryRequest 函数返回的数据的格式与下列一致：

```json
{
    "path": "/device/points?deviceId=123456",
    "method": "GET",
    "headers": "{\"Content-Type\":\"application/json\"}",
    "postParams": "{\"deviceId\":\"123456\"}"
}
```

字段说明如下：

| 字段         | 类型     | 含义 |
|--------------|----------|------|
| path         | String   | 请求路径。用于指定查询设备功能点的接口地址，例如 `/device/points?deviceId=xxx`。其中 `deviceId` 为目标设备的唯一标识，通过 query 参数拼接到 path 中。 |
| method       | String   |  HTTP 请求方法，支持 `GET` 和 `POST`，用于指定调用发现接口时使用的 HTTP 请求方式。 |
| headers      | String   | 请求头，**JSON 字符串格式**（例如：`{"Content-Type":"application/json"}`） |
| postParams   | String   | POST 请求体参数，**JSON 字符串格式**（例如：`{"deviceId":"123456"}`）。 |

:::tip
请根据设备官方技术文档或联系设备技术支持获取 `path`、`method`、`headers` 及 `postParams`。
:::

### 函数示例

```javascript
function buildDiscoverPointsRequest(deviceId) {
    return {
        path: `/device/points?deviceId=${deviceId}`,
        method: "GET",
        headers: JSON.stringify({
            "Content-Type": "application/json"
        }),
        postParams: ""
    };
}
```

## parseDiscoverPointsResponse  

请在此方法中解析设备功能点（[buildDiscoverPointsRequest](#builddiscoverpointsrequest)）发现接口的返回结果，并返回包含所有功能点信息的数组。  

Aqara Studio 会自动调用此函数，您需要在返回的数组中按要求填写每个功能点的详细结构（例如 pointId、pointName 等，具体请参见[point 对象结构](#point-对象结构)）。  

通过正确实现本方法，Aqara Studio 才能识别和接入设备支持的全部功能点，从而实现功能点的自动发现与控制。

### 入参说明

本函数的入参应仅有一个，且参数名必须为 responseString，否则 Aqara Studio 无法正确传递和解析数据。

### 返回值

为确保 Aqara Studio 可以正常获取功能点信息，您需要确保 parseDiscoverPointsResponse 函数返回的数据为 [point 对象结构](#point-对象结构) 数组。

#### point 对象结构 

| 字段名称    | 说明                                                                                     |
|-----------|-------------------------------------------------------------------------------------------|
| pointId   | 功能点 ID。请咨询设备厂商获取功能点的真实 ID，以便 Aqara Studio 发现该点。                                                    |
| pointName | 您自定义的功能点名称。                                                                              |
| pointType | 对应 Aqara Spec 的 `deviceType`，具体值请参考 Aqara Spec 相关文档。                                  |
| function| 对应 Aqara Spec 的 `functionCode`，具体值请参考 Aqara Spec 相关文档。                                  |
| trait   | 对应 Aqara Spec 的 `traitCode`，具体值请参考 Aqara Spec 相关文档。                                      |
| dataType  | 功能点数据类型，支持以下取值：<ul><li><code>bool</code>：布尔型</li><li><code>numerical</code>：数值型</li><li><code>text</code>：文本型</li></ul> |
| access    | 读写类型，<ul><li>1：仅可读</li><li>2：可读可写</li></ul>                                 |

:::tip
您可以根据实际情况选择以下任意方式构造功能点的数据结构：   
- **基于 Aqara Spec**：需填写 `pointType`、`function`、`trait` 等字段，这可以将设备的功能点关联到 Aqara Spec 协议中的对应功能点。
- **通用方式**：需填写 `dataType`、`access` 等基本字段。
:::
	
#### 示例

为确保 Aqara Studio 可以正常获取设备信息，您需要确保 parseDiscoverPointsResponse 函数返回的数据的格式与下列一致：

- 基于 Aqara Spec：
    ```json
    [
        {
            "pointId": "temperature",
            "pointName": "温度",
            "pointType": "TemperatureSensor",
            "function": "Temperature",
            "trait": "CurrentTemperature"
        }
    ]
    ```

- 使用通用方法：

    ```json
    [
        {
            "pointId": "temperature",
            "pointName": "温度",
            "dataType": "numerical",
            "access": "1"
        }
    ]
    ```

### 函数示例

```javascript
function parseDiscoverPointsResponse(responseString) {
    if (!responseString) return [];
    let obj;
    try {
        obj = JSON.parse(responseString);
    } catch (e) {
        return [];
    }
    const list = obj?.reply?.data?.points;
    if (!Array.isArray(list)) return [];
    return list.map(item => ({
        pointId: item.pointId,
        pointName: item.pointName,
        pointType: item.pointType,
        function: item.function,
        trait: item.trait
    }));
}
```

## buildReadRequest  

当 Aqara Studio 需要获取设备的状态数据（例如开关状态、温度、模式等）时，会调用此函数，传入目标设备 ID（deviceId）以及需要读取的功能点 ID 数组（pointIds）。

请在该方法中返回一个请求配置对象，用于指示 Aqara Studio 如何通过 HTTP API 查询指定设备的功能点（points）的值。

### 入参说明

本函数的入参应仅有两个，且参数名必须为 deviceId（设备 ID） 和 pointIds（功能点列表），否则 Aqara Studio 无法正确发起请求。

### 返回值

为确保 Aqara Studio 可以正常获取功能点信息，您需要确保 buildReadRequest 函数返回的数据的格式与下列一致：

```json
[
    {
        "path": "/device/read/temperature",
        "method": "GET",
        "headers": "",
        "postParams": ""
    }
]
```

:::tip
这里返回的是一个数组，如果设备支持批量读取，数组只包括一个对象即可，如果设备不支持批量读取，则可以根据实际点生成多个数组返回。
:::

字段说明如下：

| 字段         | 类型     | 含义 |
|--------------|----------|------|
| path         | String   | 请求路径。用于指定查询设备功能点的接口地址。<br />如果 `method` 为 `GET` 且包含参数，可直接拼接到 path。 |
| method       | String   |  HTTP 请求方法，支持 `GET` 和 `POST`，用于指定调用发现接口时使用的 HTTP 请求方式。 |
| headers      | String   | 请求头，**JSON 字符串格式**（例如：`{"Content-Type":"application/json"}`） |
| postParams   | String   | POST 请求体参数，**JSON 字符串格式**（例如：`{"deviceId":"123456"}`）。 |

:::tip
请根据设备官方技术文档或联系设备技术支持获取 `path`、`method`、`headers` 及 `postParams`。
:::

### 函数示例

```javascript
function buildReadRequest(deviceId, pointIds) {
    if (!deviceId || !Array.isArray(pointIds) || pointIds.length === 0) return [];
    return pointIds.map(function(pointId) {
        return {
            path: `/device/read/${deviceId}/${pointId}`,
            method: "GET",
            headers: "",
            postParams: ""
        };
    });
}
```

## parseReadResponse  

本接口用于解析通过功能点读取接口（[buildReadRequest](#buildreadrequest)）获取的设备原始数据。

请在本方法中将设备返回的 responseString 转换为 Aqara Studio 可以识别的功能点数据格式。

### 入参说明

本函数的入参应仅有两个，且参数名必须为 deviceId 和 responseString，否则 Aqara Studio 无法正确传递和解析数据。

### 返回值

为了确保 Aqara Studio 能正确获取功能点的数据，您需要保证 parseReadResponse 函数返回的数据格式与下方示例一致：

```json
{
    "temperature": "22.5",
    "humidity": "60"
}
```

字段说明：每个 key 表示功能点，对应的 value 为该点的值（均为字符串类型）

### 函数示例

```javascript
function parseReadResponse(deviceId, responseString) {
    if (!responseString) return {};
    let obj;
    try {
        obj = JSON.parse(responseString);
    } catch (e) {
        return {};
    }
    const result = {};
    if (obj.data && Array.isArray(obj.data)) {
        obj.data.forEach(item => {
            for (const key in item) {
                if (Object.prototype.hasOwnProperty.call(item, key)) {
                    result[key] = String(item[key]);
                }
            }
        });
    }
    return result;
}
```

## buildWriteRequest  

当您向在 Aqara Studio 上控制设备的功能点（如开关、温度、模式等）时，Aqara Studio 会调用此方法，并传入目标设备 ID（deviceId）、要控制的功能点 ID（pointId）以及要设置的目标值（value）。

请在该方法中返回一个请求配置对象，指示 Aqara Studio 应如何通过 HTTP API 向设备下发控制指令。

### 入参说明

本函数的入参应仅有三个，且参数名必须为 deviceId（设备 ID）、pointId（功能点 ID）和 value（功能点值），否则 Aqara Studio 无法正确发起请求。

### 返回值

为确保 Aqara Studio 可以正常控制功能点，您需要确保 buildWriteRequest 函数返回的数据的格式与下列一致：

```json
{
    path: "/device/write/temperature",
    method: "POST",
    headers: "",
    postParams: "{\"value\":22.5}"
}
```

字段说明如下：

| 字段         | 类型     | 含义 |
|--------------|----------|------|
| path         | String   | 请求路径。用于指定写入设备点的接口地址。<br />如果 `method` 为 `GET` 且包含参数，可直接拼接到 path，例 |
| method       | String   |  HTTP 请求方法，支持 `GET` 和 `POST`，用于指定调用发现接口时使用的 HTTP 请求方式。 |
| headers      | String   | 请求头，**JSON 字符串格式**（例如：`{"Content-Type":"application/json"}`） |
| postParams   | String   | POST 请求体参数，**JSON 字符串格式**（例如：`{"value":22.5}`）。 |

:::tip
请根据设备官方技术文档或联系设备技术支持获取 `path`、`method`、`headers` 及 `postParams`。
:::

### 函数示例

```javascript
function buildWriteRequest(deviceId, pointId, value) {
    if (!deviceId || !pointId) return {};
    return {
        path: `/device/write/${deviceId}/${pointId}`,
        method: "POST",
        headers: JSON.stringify({
            "Content-Type": "application/json"
        }),
        postParams: JSON.stringify({ value: value })
    };
}
```

## parseWriteResponse 

本接口用于解析功能点写入（控制）接口（[buildWriteRequest](#buildwriterequest)）返回的数据，用于判断设备的控制指令是否执行成功。

当 Aqara Studio 向设备发起写入请求（例如开关设备、调节温度等）并接收到设备返回结果时，会调用此函数，根据返回值，判断控制是否成功。

请在本方法中解析设备返回的数据，并将其转换为布尔值，便于 Aqara Studio 判断控制指令是否执行成功。

### 入参说明

本函数的入参应仅有两个，且参数名必须为 pointId 和 responseString，否则 Aqara Studio 无法正确传递和解析数据。

## 返回值

请确保函数返回值为布尔值，布尔值说明如下：
- true：功能点写入成功
- false：功能点写入失败或设备返回异常

### 函数示例

```javascript
function parseWriteResponse(pointId, responseString) {
    if (!responseString) return false;
    let obj;
    try {
        obj = JSON.parse(responseString);
    } catch (e) {
        return false;
    }
    return obj.result === "success";
}
```

## parseReportRequest 


当设备主动向 Aqara Studio 上报数据时（例如设备状态变化、传感器上报数据等），Aqara Studio 会调用此函数，从设备主动上报的数据中提取设备 ID 和各个功能点的上报数据，用于更新设备状态。

请在本方法中将设备返回的 responseString 转换为 Aqara Studio 可以识别的功能点数据格式。

### 入参说明
	
本函数的入参应仅有两个，且参数名必须为 path 和 responseString，否则 Aqara Studio 无法正确传递和解析数据。

### 返回值

返回值的格式应为 Pair\<List\<Map\<String, String>>, String>，即点信息列表和 responseMsg。

为了确保 Aqara Studio 能正确获取功能点的数据，您需要保证 parseReportResponse 函数返回的数据格式与下方示例一致：

```json
{
    deviceId: "123456",
    points: [
        { 
            pointId: "temperature", 
            value: "22.5" 
        }
    ],
    responseMsg: ""
}
```

### 函数示例

```javascript
function parseReportRequest(path, requestString) {
    if (!requestString) return { deviceId: "", points: [], responseMsg: "" };
    let obj;
    try {
        obj = JSON.parse(requestString);
    } catch (e) {
        return { deviceId: "", points: [], responseMsg: "" };
    }
    const deviceId = obj.deviceld || obj.deviceId || "";
    const points = [];
    const parans = obj.data && obj.data.parans ? obj.data.parans : {};
    for (const pointId in parans) {
        if (Object.prototype.hasOwnProperty.call(parans, pointId)) {
            const item = parans[pointId];
            points.push({
                pointId: pointId,
                value: String(item.value)
            });
        }
    }
    return { deviceId: deviceId, points: points, responseMsg: "" };
}
```
