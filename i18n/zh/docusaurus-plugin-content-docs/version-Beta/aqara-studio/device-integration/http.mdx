# HTTP 设备接入

本文说明如何将 HTTP 设备接入 Aqara Studio。

## 前提条件

在开始之前，请确保待接入的 HTTP 设备与 Aqara Studio 处于同一局域网。

## 添加 HTTP 设备和功能点

1. 在左侧侧边栏点击进入**设备管理**页面。
2. 在左侧设备树单击“+”按钮，在弹窗中选择 `HTTP`，以添加一个 HTTP 节点。
3. 选择该 HTTP 节点，进入管理界面。
4. 进入`协议配置` 选项卡，配置待接入 HTTP 设备的以下参数。

    | 参数 | 说明 |
    | -- | -- |
    | 使用 TLS | 是否对 HTTP 通信使用 TLS 加密 |
    | IP地址 | 待接入 HTTP 设备的网关 IP 地址。 |
    | 端口 | 设备用于接收和处理 HTTP 请求的网络端口，平台需通过该端口与设备进行通信。 |
    | 启用/禁用该网络 | 禁用后该网络下的设备将不可用。 |

5. 切换至 **脚本** 选项卡，按照 [JavaScript 脚本（适用于 HTTP）](./../javascript-for-http-integration.mdx) 的规范编写 JS 脚本，以指导 Aqara Studio 如何发现设备、添加与控制功能点等。
6. 切换到 **添加设备** 选项卡，点击 **发现设备** 按钮，Aqara Studio 会根据您编写的脚本自动扫描并识别可接入的 HTTP 设备，被发现的设备会在**发现设备列表**中展示。
7. 选中目标设备，点击中间的 **+** 按钮，即可将该设备添加至右侧的**已添加设备列表**，并会实时同步显示在左侧设备树的 HTTP 节点下。

    在左侧设备树展开该设备，即可查看其已识别的全部功能点。

## 多网关场景下的 HTTP 设备接入

每个 HTTP 节点仅能对应一个网关下的 HTTP 设备。如果需要接入多个不同网关下的 HTTP 设备，请在左侧设备树中多次点击“+”按钮，分别新增多个 HTTP 节点，并为每个节点单独配置相应的网关信息。请勿通过修改已有节点的网络参数来接入其他网关下的设备。
