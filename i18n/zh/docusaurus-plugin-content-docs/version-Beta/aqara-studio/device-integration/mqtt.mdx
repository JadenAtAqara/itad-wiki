# MQTT 设备接入

本文说明如何将 MQTT 设备接入 Aqara Studio。

## 前提条件

在开始之前，请确保待接入的 MQTT 设备已接入 1 个 MQTT Broker。

## 添加 MQTT 设备和功能点

1. 在左侧侧边栏点击进入**设备管理**页面。
2. 在左侧设备树单击“+”按钮，在弹窗中选择 `MQTT`，以添加 MQTT。
3. 选择 MQTT，进入管理界面。
4. 进入`添加设备`选项卡，配置待接入 MQTT 设备的以下参数，并单击 **保存并继续** 按钮。

    | 参数 | 是否必填 | 说明 |
    | -- | -- | -- |
    | 设备名称 | ✔ | 待接入 MQTT 设备的名称 |
    | 用户名 | ✘ | MQTT 服务器的用户名 |
    | 密码 | ✘ | MQTT 服务器的密码 |
    | 代理服务器端点 | ✔ | 填写该设备所连接的 MQTT Broker 地址。 |
    | 代理服务器端口 | ✔ | 填写设备所连接的 MQTT Broker 端口号。 |
    | 连接类型 | ✘ | MQTT 客户端连接时的认证/安全类型，支持以下：<ul><li>匿名</li><li>通过 SSL 匿名</li><li>通过 SSL 用户登录</li><li>用户登录无 SSL</li></ul> |

5. 请根据实际设备，参照 [JavaScript 功能点添加脚本](./../javascript-script-for-adding-data-points.mdx) 编写功能点添加脚本，完成后点击 **保存并继续**。
6. 单击 **发现功能点** 按钮，Aqara Studio 会根据脚本自动发现设备的功能点。
7. 在 **发现功能点列表** 勾选需要的点，单击中间 **+** 按钮，新增功能点会出现在右侧的**已添加功能点**列表，并同步显示在左侧设备树的相关设备下。

    :::tip
    - 如果 Aqara Studio 发现的功能点信息有误，请在该点一行单击 **添加** 按钮，即可修改信息。
    - 如果本系统没有发现你需要的功能点，您也可以 [手动添加该功能点](#如何处理-aqara-studio-无法发现-mqtt-设备功能点的问题)。
    :::

8. 单击 **完成添加** 按钮，即可完成设备和功能点添加流程。

## 常见问题

### 如何处理 Aqara Studio 无法发现 MQTT 设备功能点的问题

若 Aqara Studio 未发现您需要的功能点，请在“添加功能点”的页面单击“手动添加”按钮，在“添加点”弹窗中传入点的各项参数，完成手动添加点。

具体参数说明如下表所示：

| 参数 | 是否必填 | 说明 |
| --- | --- | --- |
| Friendly Name | ✔ | 功能点 ID。请咨询设备厂商获取功能点的真实 ID，以便 Aqara Studio 发现该点。 |
| Point Name | ✘ | 您自定义的功能点名称。 |
| Point Type | ✘ | 对应 Aqara Spec 的 `deviceType`，具体值请参考 Aqara Spec 相关文档。 |
| Function Id | ✘ | 对应 Aqara Spec 的 `functionId`，具体值请参考 Aqara Spec 相关文档。 |
| Trait Id | ✘ | 对应 Aqara Spec 的 `traitId`，具体值请参考 Aqara Spec 相关文档。 |
| Data Type | ✘ | 功能点数据类型，支持以下取值：<ul><li><code>bool</code>：布尔型</li><li><code>numerical</code>：数值型</li><li><code>text</code>：文本型</li></ul>  |
| Access | ✘ | 读写类型，<ul><li>1：仅可读</li><li>2：可读可写</li></ul> |
| Sub Topic | ✘ | Aqara Studio 通过订阅该 Topic 获取功能点的最新数据。具体 Topic 请根据设备协议或厂商文档填写。 |
| Pub Topic | ✘ | Aqara Studio 通过向此 Topic 发送消息数据控制功能点。具体 Topic 请根据设备协议或厂商文档填写。（对于仅可读的功能点，此字段留空） |
