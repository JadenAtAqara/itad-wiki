# AI 原生架构

Aqara Studio 的技术基础建立于**一体化 AI 原生架构**之上，该架构由 **本体论层、图层、技能层、遥测层** [四层紧密耦合的层级](#四层紧密耦合的技术底座)组成，同时对外呈现 **Studio Core（核心层）、Web（交互层）、Plugin（插件层）** [三层产品形态](#三层产品形态)，四层技术底座为三层产品形态提供核心能力支撑，形成 “底层技术闭环、上层能力开放” 的架构体系，确保系统的稳定性、扩展性与可定制性。

## 四层紧密耦合的技术底座

四层技术底座构成系统的 “智能内核”，实现物理环境的 **感知、推理、行动、持续优化**，各层级相互联动、数据互通，形成不可分割的整体。

- **本体论层（语义定义）**：系统的核心与基础，以 ORAP（对象、关系、动作、策略）为核心定义统一的空间语义模型，将异构设备、协议及数据抽象为一致的、机器可理解的表示形式，让系统基于房间、区域、功能及意图等空间概念运作，而非依赖设备层面的基础元素。
- **图层（状态与结构）**：即空间图谱，是本体论的具象化载体，作为空间结构、设备拓扑、实时状态及操作约束的唯一真实数据来源，在设备层、边缘层及云端层之间提供一致的、可查询的及可订阅的模型，确保全系统的一致性、可追溯性及可解释性。
- **技能层（受管辖的执行）**：系统的 “行动入口”，所有操作均通过标准化技能执行，技能在明确的策略约束下封装控制、自动化及配置功能，此受管辖的执行模型确保所有操作均安全、可审计且可逆。
- **遥测层（反馈与学习）**：系统的 “进化引擎”，捕捉系统交互的完整生命周期 —— 从意图与上下文到执行与结果，从而实现持续反馈、系统优化及 AI 驱动的学习，通过该闭环机制，系统随时间持续适应并提升其性能。

## 三层产品形态

基于四层技术底座，对外呈现三层易操作、可扩展的产品形态，为用户提供从核心能力到交互操作、功能扩展的全流程支撑：
1. **Studio Core（核心层）**：产品的 “算力大脑”，承载本体语义建模、空间图谱构建、协议解析、逻辑运算、数据处理、设备管控等核心能力，为上层交互与插件提供统一的能力支撑，保障系统核心能力的稳定性与一致性。
2. **Plugin（插件层）**：系统的能力扩展载体，支持自由组合与配置，包含标准内置插件（Aqara 官方）、持续迭代公共插件（Aqara 官方）、客户定制私有插件（Aqara / 客户联合开发）、Builder 共建插件（社区开发者）四类，适配不同场景的个性化需求。
3. **Web（交互层）**：基于 B/S 架构的浏览器端交互界面，无需安装客户端，支持多终端访问，为用户提供空间建模、设备管理、自动化创作、运维监控、系统管理等全流程操作能力，是用户与核心层的交互入口。
