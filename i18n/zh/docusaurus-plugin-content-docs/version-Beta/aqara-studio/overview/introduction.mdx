             
# 平台概述

## 什么是Aqara Studio？

Aqara Studio 是**基于空间本体论（Spatial Ontology）构建的本体原生空间智能操作系统**，核心是将真实物理空间数字化，为房间、楼宇、跨地域网点等各类物理空间装上 “智能大脑”。

产品以**本地可靠性**、**生态可扩展性**、**云端协同性**为核心特征，依托**本体论层**、**图层**、**技能层**、**遥测层**四层紧密耦合的 [AI 原生架构](./ai-native-architecture.mdx)，打造 “感知 - 推理 - 行动 - 优化” 的闭环系统，打破设备、协议、系统之间的壁垒，将空间智能贯彻于智能家居、商业楼宇、智慧办公、跨地域网点协同等全场景，为渠道合作伙伴、行业集成商、开发者提供灵活、可定制、可扩展的空间智能落地解决方案。

作为**本体原生的空间智能操作系统**，与传统 IoT 平台 / 智能家居配置工具、传统楼宇自控系统相比，Aqara Studio在技术架构、核心逻辑、能力边界等方面存在本质差异，即**从 “设备管控”/“协议联动” 升级为 “空间智能”**，通过本体论实现空间与设备的解耦，打破了场景、协议、系统的壁垒，为多场景提供一体化的空间智能解决方案。

## 系统架构

Aqara Studio 采用了分布式与集中式相结合的系统架构设计，能够根据不同网络需求，灵活支持多层级分布式组网解决方案。该系统支持云端、边缘端及终端设备的联合部署，确保在不同网络环境下实现无缝的数据互联和服务多元化应用需求。

- **分布式与集中式结合**：混合架构既保持了分布式系统的灵活性和高可用性，又可通过集中式管理简化了操作和维护流程。
- **多层级分布式组网**：多层分布式网络配置可根据项目的规模和复杂度进行定制化部署。
- **云、边、端融合部署**：Aqara Studio 能在云端与边缘端，提供提供统一的管理的集成方案，确保数据处理和传输的高效性和安全性。
- **满足多样化需求**：在大型园区、智能楼宇以及家庭环境中，Aqara Studio 可在不同的网络环境下，实现从基础连接到高级数据分析的各种应用场景需求。

## 核心优势

Aqara Studio 的核心竞争力源于**本体原生的技术架构**与**全场景适配的产品设计**，相较于传统 IoT 平台、楼宇自控工具，从底层逻辑上实现了技术与应用的升级，核心优势体现在六大维度：

### 本地闭环 + 云端协同，安全与灵活兼具

核心逻辑、设备自动化、空间模型均支持**本地化运行**，不依赖外网，保障数据安全与运行稳定性；同时支持云端连接，实现远程访问、固件升级、增值服务拓展，兼顾本地可靠性与云端灵活性。

### 多协议融合，全生态无缝兼容

原生支持物联网行业主流**生态协议**（AqaraHub、Matter、Home Assistant）、**工业通信协议**（KNX IP、BACnet IP、Modbus TCP、Modbus Serial、DALI（via Modbus TCP）、Siemens PPI）、**通用 IoT 协议**（Zigbee、Thread、Bluetooth、Wi-Fi（2.4G/5G）、MQTT、OPC UA、CoAP、TCP/UDP），实现异构设备、新旧系统的无缝接入与互联互通，无需额外协议转换工具，兼容传统 legacy 系统与新一代智能设备。

### 本体原生，空间与设备深度解耦

基于 [空间本体论与 ORAP 语义模型](./core-concepts.mdx#空间本体论与-orap-模型) 构建统一语义体系，系统围绕 “空间” 而非 “设备” 运作，应用执行与物理设备完全解耦，无需关注底层设备品牌、协议差异，大幅降低场景适配与二次开发成本。

### 分布式架构，多形态部署适配全场景

支持网关融合、边端部署、云端运行三种部署形态，可单独 / 混合部署，满足别墅家庭本地零延迟控制、办公楼宇集中管控、跨地域网点远程协同等不同场景的管控、网络、数据安全需求，横向可无限扩展。

### 插件化架构，产品能力可生长、可定制

采用 “核心稳定 + 插件扩展” 的设计理念，支持官方插件、定制私有插件、社区共建插件，可根据业务需求灵活扩展功能，适配从家庭到大型商业项目的全生命周期需求。

### 低代码 + 深度开放，开发门槛全覆盖

提供**IFTTT**/**Flow**/**Logic**可视化逻辑编排工具，实现零代码 / 低代码开发；同时开放 **OpenAPI**、[MCP 协议](./core-concepts.mdx#mcp-协议)、插件开发框架，满足专业集成商的深度定制需求，覆盖普通开发者到行业专家的全层级开发需求。
