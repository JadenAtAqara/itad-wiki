# JavaScript 脚本（适用于 MQTT）

本文件介绍了如何通过 JavaScript 脚本在 Aqara Studio 中自定义添加 MQTT 设备功能点及其解析方法。

## 脚本介绍

本脚本**必须**包含以下 4 个 JavaScript 方法：
- [parseDiscoveryResponse](#parsediscoveryresponse)
- [parseDiscoverPointsResponse](#parsediscoverpointsresponse)
- [parseReportRequest](#parsereportrequest)
- [buildWriteRequest](#buildwriterequest)

:::tip
请勿更改方法的参数及其输入输出结构。
:::

## parseDiscoveryResponse

该方法 `parseDiscoveryResponse` 主要用于向 Aqara Studio 声明需要接入的 MQTT 设备的基本配置信息。

您需要以**对象数组**的形式返回待接入设备的基本信息，详情请参考 [设备对象结构](#设备对象结构)。

Aqara Studio 会调用此方法以发现所有可用设备，实现自动接入与管理。

### 设备对象结构

| 字段名称 | 说明 |
| --- | --- |
| deviceName | 设备名称。 |
| deviceId   | 设备 ID。 |
| brokerEndpoint | MQTT Broker 的地址。 |
| brokerPort | MQTT Broker 的端口。 |
| connectionType | 连接类型。支持以下取值：<ul><li>0：匿名</li><li>1：通过 SSL 匿名</li><li>2：通过 SSL 用户登录</li><li>3：用户登录无 SSL</li></ul> |
| username | 用户名。 |
| password | 密码。 |

### 示例 

```javascript
function parseDiscoveryResponse() {
	return [
		{
			deviceName: "TestDevice",
			deviceId: "98CC4D189E60",
			brokerEndpoint: "127.0.0.1",
			brokerPort: 1883,
			connectionType: 3,
			username: "admin",
			password: "public"
		}
	];
}
```

## parseDiscoverPointsResponse

该方法负责向 Aqara Studio 声明“该设备具备哪些功能点”。

您需要在返回数组中填写一个或多个待发现功能点结构（如 `pointId` 等，详情请见 [point 对象结构](#point-对象结构)）。这样，Aqara Studio 才能在功能点发现流程中会调用该函数，解析返回值，从而识别设备具备的所有功能点，实现功能点无缝接入和控制。

### point 对象结构

| 字段名称    | 说明                                                                                     |
|-----------|-------------------------------------------------------------------------------------------|
| pointId   | 功能点 ID。请咨询设备厂商获取功能点的真实 ID，以便 Aqara Studio 发现该点。                                                    |
| pointName | 您自定义的功能点名称。                                                                              |
| pointType | 对应 Aqara Spec 的 `deviceType`，具体值请参考 Aqara Spec 相关文档。                                  |
| functionCode | 对应 Aqara Spec 的 `functionCode`，具体值请参考 Aqara Spec 相关文档。                                  |
| traitCode   | 对应 Aqara Spec 的 `traitCode`，具体值请参考 Aqara Spec 相关文档。                                      |
| dataType  | 功能点数据类型，支持以下取值：<ul><li><code>bool</code>：布尔型</li><li><code>numerical</code>：数值型</li><li><code>text</code>：文本型</li></ul> |
| access    | 读写类型，<ul><li>1：仅可读</li><li>2：可读可写</li></ul>                                 |
| subTopic  | （**仅适用于 MQTT 设备**）Aqara Studio 通过订阅该 Topic 获取功能点的最新数据。具体 Topic 请根据设备协议或厂商文档填写。  |
| pubTopic  | （**仅适用于 MQTT 设备**）Aqara Studio 通过向此 Topic 发送消息数据控制功能点。具体 Topic 请根据设备协议或厂商文档填写。（对于仅可读的功能点，此字段留空）        |

:::tip
您可以根据实际情况选择以下任意方式构造功能点的数据结构：   
- **基于 Aqara Spec**：需填写 `pointType`、`functionCode`、`traitCode` 等字段，这可以将设备的功能点关联到 Aqara Spec 协议中的对应功能点。
- **通用方式**：需填写 `dataType`、`access` 等基本字段。
:::

### 示例

```javascript
function parseDiscoverPointsResponse() {
	return [
		// 基于 Aqara Spec 接入功能点
		{
			pointId: "1#冷机运行状态",
			pointName: "1#冷机运行状态",
			pointType: "AirConditioner",
			functionCode: "Output",
			traitCode: "OnOff",
			subTopic: "/WEI-iEdge/realdata/7a1dd258f32940d95a542c68",
			pubTopic: "/WEI-iEdge/write/req/7a1dd258f32940d95a542c68"
		},
		// 基于通用方式接入功能点
		{
			pointId: "1#冷机关机/开机读",
			pointName: "1#冷机关机/开机读",
			dataType: "bool",
			access: 1,
			subTopic: "/WEI-iEdge/realdata/7a1dd258f32940d95a542c68",
			pubTopic: "/WEI-iEdge/write/req/7a1dd258f32940d95a542c68"
		},
		{
			pointId: "1#冷机出水温度设定读",
			pointName: "1#冷机出水温度设定读",
			dataType: "numerical",
			access: 2,
			subTopic: "/WEI-iEdge/realdata/7a1dd258f32940d95a542c68",
			pubTopic: "/WEI-iEdge/write/req/7a1dd258f32940d95a542c68"
		},
		{
			pointId: "1#冷机制冷量百分比",
			pointName: "1#冷机制冷量百分比",
			dataType: "numerical",
			access: 2,
			subTopic: "/WEI-iEdge/realdata/7a1dd258f32940d95a542c68",
			pubTopic: "/WEI-iEdge/write/req/7a1dd258f32940d95a542c68"
		}
	];
}
```

## parseReportRequest

该方法负责从设备上报的数据解析出指定功能点（`pointId`）的当前值。

当 Aqara Studio 收到设备上报消息时，会调用此函数并传入功能点 ID 和原始消息字符串。您需要在本函数中实现从其中的 `values` 字段提取出对应 `pointId` 的值。这样 Aqara Studio 才能实时更新与展示功能点值。

:::tip
请结合设备的实际数据上报协议，自定义解析逻辑，确保能准确从原始消息中提取 `pointId` 对应的数值（例如示例中的 `obj["values"][pointId]`）。
:::

### 示例

```javascript
// 以下为解析 MQTT 设备上报 JSON 消息字符串的示例
function parseReportRequest(pointId, jsonStr) {
	try {
		const obj = JSON.parse(jsonStr);
		return obj["values"][pointId]?.toString();
	} catch (e) {
		return null;
	}
}
```

## buildWriteRequest

该方法负责根据要控制的功能点 ID 和设定的值，构造向设备下发控制指令的字符串。

当 Aqara Studio 需要向设备发送控制命令（如开关、设定温度）时，会调用此函数，传入目标功能点 ID 和要设置的数值。函数需返回一个字符串，该字符串需符合设备协议的控制指令格式，通常包含设备标识、时间戳和具体的值字段。

请根据设备实际可接受的报文格式，在函数内部合理构建对象结构。例如，以 MQTT 设备为例，可包含 `node`、`group`、`timestamp` 等字段，并将 `pointId` 对应的值被写入 `values` 字段。

### 示例

```javascript
// 以通过 JSON 字符串控制 MQTT 设备为例
// value 参数由 Aqara Studio 自动传入，无需手动声明或赋值
function buildWriteRequest(pointId, value) {
	try {
		const obj = {
			"node": "冷机",
			"group": "1#冷机",
			"timestamp": 1768204054654,
			"values": {
				
			},
			"errors": {
				
			},
			"metas": {
				
			}
		};
		obj.values[pointId] = parseFloat(value);
		return JSON.stringify(obj);
	} catch (e) {
		return null;
	}
}
```

## 完整示例

下面是一个添加 MQTT 设备功能点的完整示例：

```javascript
(function(){
	function parseDiscoveryResponse() {
		return [
			{
				deviceName: "TestDevice",
				deviceId: "98CC4D189E60",
				brokerEndpoint: "127.0.0.1",
				brokerPort: 1883,
				connectionType: 3,
				username: "admin",
				password: "public"
			}
		];
	}

	function parseDiscoverPointsResponse() {
		return [
			// 基于通用方式接入功能点
			{
				pointId: "busAddr1.on",
				pointName: "开关1",
				dataType: "bool",
				access: 2,
				subTopic: "/dev/MTS/98CC4D189E60/status/json",
				pubTopic: "/dev/MTS/98CC4D189E60/cmd/json"
			},
			{
				pointId: "busAddr2.on",
				pointName: "开关2",
				dataType: "bool",
				access: 2,
				subTopic: "/dev/MTS/98CC4D189E60/status/json",
				pubTopic: "/dev/MTS/98CC4D189E60/cmd/json"
			}
		];
	}

	function parseReportRequest(pointId, jsonStr) {
		try {
			const obj = JSON.parse(jsonStr);
			const match = /^busAddr(\d+)\.on$/.exec(pointId);
			if (!match || !Array.isArray(obj.data)) {
				return null;
			}

			const busAddr = parseInt(match[1], 10);
			for (const item of obj.data) {
				const state = item && item.mcbComSta;
				if (state && Number(state.busAddr) === busAddr && state.on !== undefined && state.on !== null) {
					return String(state.on);
				}
			}
			return null;
		} catch (e) {
			return null;
		}
	}

        // value 参数由 Aqara Studio 自动传入，无需手动声明或赋值。
        function buildWriteRequest(pointId, value) {
		try {
			const match = /^busAddr(\d+)\.on$/.exec(pointId);
			if (!match) {
				return null;
			}

			const busAddr = parseInt(match[1], 10);
			const isOn = value === true || value === 1 || value === "1" || value === "true" || value === "on";
			const obj = {
				method: 3,
				dstID: 43008827855272960,
				msgCode: 120,
				timeMS: 0,
				data: [
					{
						tag: 305,
						cmd: isOn ? "do_turn_on" : "do_turn_off",
						busAddr: [busAddr]
					}
				]
			};
			return JSON.stringify(obj);
		} catch (e) {
			return null;
		}
	}

    globalThis.parseDiscoveryResponse = parseDiscoveryResponse;
    globalThis.parseDiscoverPointsResponse = parseDiscoverPointsResponse;
    globalThis.buildWriteRequest = buildWriteRequest;
    globalThis.parseReportRequest = parseReportRequest;
	return null	
})()
```
