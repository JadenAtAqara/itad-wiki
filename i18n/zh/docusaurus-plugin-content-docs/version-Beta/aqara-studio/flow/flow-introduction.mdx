# 高阶 Flow

通过灵活的可视化配置能力，您可以基于各类卡片组成数据流逻辑，快速构建适用于多种场景的自动化流程，轻松实现物联网设备的智能控制与协同运行。

## 新建自动化

如需了解如何新建高阶 Flow，请参考 [自动化管理 - 新建自动化](./../automation.mdx#新建自动化)。

此时，您可以在画布上添加并串联各类[卡片](#卡片)，构建适用于特定场景的设备联动逻辑。

## 卡片

在 Aqara Studio 中，卡片在自动化中扮演着“信息中枢”和“控制枢纽”的角色。它将后台复杂的自动化逻辑（如“有人且光线暗就开灯”）转化为清晰可视的设备状态和一键触发的场景开关，让无形的自动化变得直观、可控。

### 组成要素

卡片包含以下要素：

| 要素 | 说明 |
| -- | -- |
| 输入点 | 当卡片拥有输入点时，表示其可被触发或可接受数据。 |
| 参数 | 可用于设置需要关注的状态、执行的动作或其他操作。|
| 输出点 | 当卡片拥有输出点时，表示其可触发其他步骤或传出数据。 |

#### 连接点类型

在 Aqara Studio 中，输出点和输入点都可以为圆点或三角形，它们代表的含义不同：

| 符号 | 名称 | 功能 | 触发机制 | 典型示例 |
|:---:|:---:|:---:|:---:|:---:|
| ● | 数据流的连接点 | 在卡片之间传递数据。 | 默认触发卡片的执行。数据的变化或到达会驱使卡片运行。 | 将温度传感器的温度 [赋值变量](./../flow/variables-blocks.mdx#赋值变量) 卡片与 [数值运算](./../flow/other-blocks.mdx#数值运算) 卡片相连。每当温度传感器监测到新的温度，就会通过此点自动将该数值传递给 `数值运算` 卡片并触发计算。 |
| ▷| 事件流的连接点 | 表示一个瞬时的信号或动作，控制卡片的执行时机。 | 当事件发生时，通知卡片开始执行。 | 一个按钮的点击动作（事件流）触发一个开关卡片。 |

Aqara 建议您在连接卡片时保持连接点的类型一致，即“圆形对圆形，三角对三角”。
