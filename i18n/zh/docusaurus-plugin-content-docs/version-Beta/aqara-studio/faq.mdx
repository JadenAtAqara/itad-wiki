# 常见问题

## Aqara Studio 无法登录如何处理？

请根据您是 [本地用户](#本地用户) 还是 [远程用户](#远程用户)，查看如下排查指引。

#### 本地用户

1. **确认账号是否存在：**
    - 若没有账号，请联系具有 `超级管理员` 权限的本地用户为您开通账号。
    - 若已有账号，参考以下信息回想起您的用户名和密码：
        - 常用用户名包括：`super`（即 `超级管理员`）、`admin`、`operator`，或您自定义设置的其他用户名。
        - 密码要求：8-16 位，需同时包含大小写字母、数字和特殊字符。
2. **如仍无法想起密码，请根据用户角色，选择以下方式：**
    - `超级管理员`：请根据 Aqara Studio 的运行形态，选择以下方式：
        - **网关融合形态**：请参考设备说明书操作恢复出厂设置。
        - **边端部署形态**：请卸载当前 Aqara Studio 并重新进行安装。
        :::tip
        恢复出厂设置将清除所有数据。
        :::
    - 其他角色：请联系 `超级管理员` [重置密码](./user.mdx#重置本地用户密码)。

#### 远程用户

如远程用户在登录界面输入 Aqara Builder 账号和密码后提示登录失败，请无需继续尝试，此为错误的登录方式。

正确方式应为：先登录 [Aqara Builder](https://builder.aqara.com/)，在左侧菜单的 **Studios** 中找到有权限访问的 Studio，点击对应的 **Connect** 按钮，即可远程访问该 Aqara Studio。

## 在 Windows 系统无法访问 Aqara Studio 本机访问域名时如何处理？


当您在 Windows 操作系统上无法访问 Aqara Studio 本机访问域名出现异常时，可能是因为 DNS 无法解析该域名。您可以参考任意方式配置 DNS。

#### 通过图形界面进行配置

1. 点击菜单栏，搜索“设置”，进入“网络和 Internet 设置”。
2. 在左侧菜单中选择“以太网”或“Wi-Fi”，根据您当前的网络连接方式进行选择。
3. 点击已连接的网络名称，进入网络属性页面。
4. 下拉找到“IP 分配”部分，点击“编辑”按钮。
5. 在弹出的窗口中，将“DNS 服务器设置”选择为手动，并开启 IPv4。
6. 仅填写首选 DNS 服务器为 `224.0.0.251`，请将其他 DNS 地址删除后保存设置。

:::tip
- 如您使用有线网络（以太网），配置步骤与 Wi-Fi 相同。
- 通过上述方式，无需修改注册表即可完成 DNS 配置，适合在局域网限制外网时使用。
:::
