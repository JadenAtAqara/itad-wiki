# Log Management

Select **Log Audit** in the sidebar to access the following four types of logs: [Device Log](#device-log), [Link Log](#link-log), [User Operation Log](#user-operation-log) and [System Update Log](#system-update-log). These logs provide detailed data to support system operation and security audits.

- **Device Log**: The device log records all changes in the function point values of devices, allowing you to track device status and operation history.
- **Link Log**: The link log includes the connection status and target IP information for Aqara Studio and various protocols.
- **User Operation Log**: The user operation log comprehensively records user logins and operation behaviors, making it easy to trace activities and clarify responsibilities.
- **System Update Log**: The system update log records information related to system upgrades, helping to trace the update history.

All logs support time filtering and keyword search, meeting daily operation and audit requirements.

:::tip
By default, Aqara Studio displays logs from 00:00:00 to 23:59:59 of the current day. To view historical logs, please manually select the desired time range.
:::

## Device Log

You can view all records of changes in device function point values. You can also filter the logs by device ID, device name, and function point name.

## Link Log 

You can view whether the link channels between Aqara Studio and each protocol (such as Aqara Cloud-M300, Studio Connect-BACnet IP) are operating normally (showing the most recent 500 records). You can also filter the logs by link name.

## User Operation Log

You can view all records of user logins and function usage (showing the most recent 500 records). You can also filter the logs by user name.

## System Update Log

You can view the time, version number, and result of each system update, which helps track the update history and troubleshoot issues.
