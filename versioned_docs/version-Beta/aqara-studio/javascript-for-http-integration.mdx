# JavaScript for HTTP Integration

This document introduces how to add HTTP devices and function points as well as their parsing methods in Aqara Studio using JavaScript scripts.

## Script Introduction

**You must implement the following 9 methods in this script** so that Aqara Studio can obtain all the information needed to initiate HTTP requests and instruct how to parse the response data into a standard format recognized by Aqara Studio.

| Method | Description |
| --- | --- |
| [buildDiscoveryRequest](#builddiscoveryrequest) | Builds the request message used to initiate device discovery. |
| [parseDiscoveryResponse](#parsediscoveryresponse) | Parses the response message returned from device discovery into structured data. |
| [buildDiscoverPointsRequest](#builddiscoverpointsrequest) | Builds the request message to query device points. |
| [parseDiscoverPointsResponse](#parsediscoverpointsresponse) | Parses the response message from the device points query into structured ponit information data. |
| [buildReadRequest](#buildreadrequest) | Builds the request message to read device point data. |
| [parseReadResponse](#parsereadresponse) | Parses the response message from the device point read operation into structured data. |
| [buildWriteRequest](#buildwriterequest) | Builds the request message to write device point data. |
| [parseWriteResponse](#parsewriteresponse) | Parses the response message from the device point write operation into structured data. |
| [parseReportRequest](#parsereportrequest) | Parses device-reported messages, extracting device ID, point ID, point values, and related data. |

:::tip
Please do not modify the names and parameters of the methods.
:::

## Prerequisites

Before writing the script, please make sure you fully understand the following HTTP interfaces of the target device, including but not limited to: request endpoint, request method, request headers, request parameters, and response data structure. It is recommended to focus on the following interfaces:
- Device discovery interface
- Function point (attribute point) query interface
- Function point data read interface
- Function point data write interface
- Function point data reporting interface

Only by thoroughly understanding the above interfaces can you ensure that your JavaScript script correctly implements the HTTP protocol according to the device specifications and completes data parsing accordingly.

## buildDiscoveryRequest

Please return a request configuration object in this method to instruct Aqara Studio how to discover devices via the HTTP API.

### Parameter Description

It is recommended that this function has no parameters.

### Return Value

To ensure Aqara Studio can successfully initiate the device discovery request, the return value of the buildDiscoveryRequest function must adhere to the following format:

```json
{
    "path": "/report/discoveryDevices",
    "method": "POST",
    "headers": "{\"Content-Type\":\"application/json\"}",
    "postParams": "{\"appId\":\"V0001\"}"
}
```

Field description:

| Field      | Type   | Description |
|------------|--------|-------------|
| path       | String | Request path. Specifies the device discovery endpoint, for example `/report/discoveryDevices`.<br />If `method` is `GET` and there are parameters, they should be appended directly to the path as in `/discoveryDevices?channel=1`. |
| method     | String | HTTP request method, supports `GET` and `POST`, used to specify which HTTP method is required for the discovery endpoint. |
| headers    | String | Request headers, **in JSON string format** (e.g., `{"Content-Type":"application/json"}`). |
| postParams | String | POST request body parameters, **in JSON string format** (e.g., `{"appId":"V0001"}`). |

:::tip
Please consult the official technical documentation for your device or contact its technical support to obtain the correct values for `path`, `method`, `headers`, and `postParams`.
:::

### Example Function
    
```javascript
function buildDiscoveryRequest() {
    return {
        path: "/report/discoveryDevices",
        method: "POST",
        headers: JSON.stringify({
            "Content-Type": "application/json"
        }),
        postParams: JSON.stringify({
            "appId": "V0001"
        })
    };
}
```

## parseDiscoveryResponse  

In this method, please parse the response from the device discovery API ([buildDiscoveryRequest](#builddiscoveryrequest)), and return an array of device objects, each including device ID (`deviceId`) and device name (`deviceName`), for use by Aqara Studio.

In this method, please parse the device discovery response, and return an array of device objects, each including a device ID (`deviceId`) and device name (`deviceName`), for use by Aqara Studio.


### Parameter Description

It is recommended that this function accepts only one string parameter, which is the return result of the device discovery request.

### Return Value

To ensure Aqara Studio can successfully retrieve device information, the data returned by the `parseDiscoveryResponse` function must conform to the following format:

```json
[
    {
        "deviceId": "123456",
        "deviceName": "Temperature Sensor"
    }
]
```

### Function Example

Let's assume that the device discovery interface returns data as follows:

```json
{
    "reply": {
        "data": {
            "meta": {
                "limit": 10,
                "offset": 0
            },
            "list": [
                {
                    "deviceId": "123456",
                    "deviceName": "test1",
                    "ip": "127.0.0.1",
                    "createTime": "qwerasd"
                }
            ]
        },
        "returnCode": {
            "type": "S",
            "code": "AAAAA",
            "domain": null
        }
    }
}
```

To process the above data, you can design the function with reference to the following example:

```javascript
function parseDiscoveryResponse(responseString) {
    if (!responseString) return [];
    let obj;
    try {
        obj = JSON.parse(responseString);
    } catch (e) {
        return [];
    }
    const list = obj?.reply?.data?.list;
    if (!Array.isArray(list)) return [];
    // Return an array, each element contains deviceId and deviceName
    return list.map(item => ({
        deviceId: item.deviceId,
        deviceName: item.deviceName
    }));
}
```

## buildDiscoverPointsRequest  

Please return a request configuration object in this method to instruct Aqara Studio how to query the function points (points) of a specified device via HTTP API.


### Parameter Description

It is recommended that this function only accepts a string parameter, which is used to input the target device ID.

### Return Value

To ensure Aqara Studio can successfully initiate the request to the device, you need to make sure the buildDiscoverPointsRequest function returns data in the following format:

```json
{
    "path": "/device/points?deviceId=123456",
    "method": "GET",
    "headers": "{\"Content-Type\":\"application/json\"}",
    "postParams": "{\"deviceId\":\"123456\"}"
}
```

Field description:

| Field      | Type   | Description |
|------------|--------|-------------|
| path       | String | Request path. Specifies the endpoint for querying device function points, for example `/device/points?deviceId=xxx`. Here, `deviceId` is the unique identifier of the target device and is appended to the path as a query parameter. |
| path       | String | Request path. Specifies the endpoint for querying device function points. <br />If `method` is `GET` and there are parameters, they can be appended directly to the path, such as `/device/points?deviceId=xxx`. |
| method     | String | HTTP request method, supports `GET` and `POST`, used to specify which HTTP method is required for the query interface. |
| headers    | String | Request headers, **in JSON string format** (e.g., `{"Content-Type":"application/json"}`). |
| postParams | String | POST request body parameters, **in JSON string format** (e.g., `{"deviceId":"123456"}`). |

:::tip
Please consult the official technical documentation for your device or contact its technical support to obtain the correct values for `path`, `method`, `headers`, and `postParams`.
:::

### Example

```javascript
function buildDiscoverPointsRequest(deviceId) {
    return {
        path: `/device/points?deviceId=${deviceId}`,
        method: "GET",
        headers: JSON.stringify({
            "Content-Type": "application/json"
        }),
        postParams: ""
    };
}
```

## parseDiscoverPointsResponse  

Please parse the function point discovery response in this method and return an array containing all function point information.

Aqara Studio will automatically call this function. You need to fill in the detailed structure for each function point in the returned array (such as pointId, pointName, etc. For details, please refer to [Point Object Structure](#point-object-structure)).

By correctly implementing this method, Aqara Studio can recognize and access all supported function points of the device, enabling automatic discovery and control of these points.

### Parameter Description

It is recommended that this function only accepts one string parameter, which is used to pass the result of the function point discovery request.

### Return Value

To ensure Aqara Studio can correctly obtain function point information, you must ensure that the parseDiscoverPointsResponse function returns an array of [point object structures](#point-object-structure).


#### Point Object Structure

| Field Name | Description |
|-----------|-------------|
| pointId   | Data point ID. Please consult the device manufacturer to obtain the actual ID to allow Aqara Studio to discover this point. |
| pointName | Name for the data point defined by you. |
| pointType | Same as the `deviceType` in Aqara Spec. Please refer to Aqara Spec documentation for the specific value. |
| functionCode  | Same as the `functionCode` in Aqara Spec. Please refer to Aqara Spec documentation for the specific value. |
| traitCode     | Same as the `traitCode` in Aqara Spec. Please refer to Aqara Spec documentation for the specific value. |
| dataType  | Data point type. Supported values: <ul><li><code>bool</code>: Boolean</li><li><code>numerical</code>: Numeric</li><li><code>text</code>: Text</li></ul> |
| access    | Access mode of the resource. <ul><li>1: Read-only</li><li>2: Read-write</li></ul> |

:::tip
When adding new data points, you may choose one of the following approaches depending on your needs:

- **Based on Aqara Spec**: You need to fill in fields such as `pointType`, `functionCode`, and `traitCode`, which enables the data point to be compatible with the Aqara Spec.
- **General Approach**: You only need to fill in the basic fields such as `dataType` and `access`.
:::
	
#### Return Examples

To ensure Aqara Studio can correctly obtain device information, you need to make sure that the data returned by the parseDiscoverPointsResponse function matches one of the following formats:

- Based on Aqara Spec:
    ```json
    [
        {
            "pointId": "temperature",
            "pointName": "Temperature",
            "pointType": "TemperatureSensor",
            "functionCode": "Temperature",
            "traitCode": "CurrentTemperature"
        }
    ]
    ```

- Using the general method:

    ```json
    [
        {
            "pointId": "temperature",
            "pointName": "Temperature",
            "dataType": "numerical",
            "access": "1"
        }
    ]
    ```

### Example

```javascript
function parseDiscoverPointsResponse(responseString) {
    if (!responseString) return [];
    let obj;
    try {
        obj = JSON.parse(responseString);
    } catch (e) {
        return [];
    }
    const list = obj?.reply?.data?.points;
    if (!Array.isArray(list)) return [];
    return list.map(item => ({
        pointId: item.pointId,
        pointName: item.pointName,
        pointType: item.pointType,
        functionCode: item.functionCode,
        traitCode: item.traitCode
    }));
}
```

## buildReadRequest  

When Aqara Studio needs to obtain status data (such as switch state, temperature, mode, etc.) from a device, it will invoke this function, passing in the target device ID (`deviceId`) and an array of the function point IDs to be read (`pointIds`).

In this function, please return a request configuration object that instructs Aqara Studio how to query the values of the specified device's function points via the HTTP API.


### Parameter Description

It is recommended that this function accepts two string parameters to pass in the device ID and the list of function points.


### Return Value

To ensure Aqara Studio can properly obtain function point information, you need to make sure the buildReadRequest function returns data in the following format:

```json
[
    {
        "path": "/device/read/temperature",
        "method": "GET",
        "headers": "",
        "postParams": ""
    }
]
```

:::tip
This function should return an array. If the device supports batch reading, the array can contain just one object. If the device does not support batch reading, you can generate multiple objects in the array based on the real points.
:::

Field description:

| Field      | Type   | Description |
|------------|--------|-------------|
| path       | String | Request path. Specifies the endpoint for querying device function points. <br />If `method` is `GET` and there are parameters, you may append them directly to the path. |
| method     | String | HTTP request method. Supports `GET` and `POST` and indicates the HTTP method to use for invoking the query. |
| headers    | String | Request headers, **in JSON string format** (for example: `{"Content-Type":"application/json"}`) |
| postParams | String | POST request body parameters, **in JSON string format** (for example: `{"deviceId":"123456"}`). |

:::tip
Please consult the official technical documentation for your device or contact technical support to obtain the correct values for `path`, `method`, `headers`, and `postParams`.
:::

### Example

```javascript
function buildReadRequest(deviceId, pointIds) {
    if (!deviceId || !Array.isArray(pointIds) || pointIds.length === 0) return [];
    return pointIds.map(function(pointId) {
        return {
            path: `/device/read/${deviceId}/${pointId}`,
            method: "GET",
            headers: "",
            postParams: ""
        };
    });
}
```

## parseReadResponse  


This method is used to parse the result of the function point read request (raw data).

In this function, please convert the raw data returned by the device into a function point data format that Aqara Studio can recognize.

### Parameter Description

It is recommended that this function accepts two string parameters, which are used to pass in the device ID and the list of function points.


### Return Value

To ensure Aqara Studio can correctly obtain the function point data, you need to make sure that the data returned by the parseReadResponse function conforms to the following format:

```json
{
    "temperature": "22.5",
    "humidity": "60"
}
```

Field description: Each key represents a function point, and the corresponding value is the value of that point (all as strings).

### Example

```javascript
function parseReadResponse(deviceId, responseString) {
    if (!responseString) return {};
    let obj;
    try {
        obj = JSON.parse(responseString);
    } catch (e) {
        return {};
    }
    const result = {};
    if (obj.data && Array.isArray(obj.data)) {
        obj.data.forEach(item => {
            for (const key in item) {
                if (Object.prototype.hasOwnProperty.call(item, key)) {
                    result[key] = String(item[key]);
                }
            }
        });
    }
    return result;
}
```

## buildWriteRequest  

When you need to control a device function point (such as switch, temperature, mode, etc.) in Aqara Studio, Aqara Studio will call this method with the target device ID (`deviceId`), the function point ID to control (`pointId`), and the value to be set (`value`).

In this method, please return a request configuration object to instruct Aqara Studio how to send a control command to the device via the HTTP API.

### Parameter Description

It is recommended that this function accepts three string parameters, which are used to pass in the device ID, function point ID, and function point value.


### Return Value

To ensure Aqara Studio can properly control the function point, the object returned by the buildWriteRequest function must be in the following format:

```json
{
    path: "/device/write/temperature",
    method: "POST",
    headers: "",
    postParams: "{\"value\":22.5}"
}
```

Field descriptions:

| Field        | Type    | Description |
|--------------|---------|-------------|
| path         | String  | Request path. Specifies the endpoint to write to the device function point.<br />If `method` is `GET` and includes parameters, you can append them directly to the path. |
| method       | String  | HTTP request method. Supports `GET` and `POST`, used to specify the HTTP method for the command. |
| headers      | String  | Request headers, **in JSON string format** (e.g., `{"Content-Type":"application/json"}`) |
| postParams   | String  | POST request body parameters, **in JSON string format** (e.g., `{"value":22.5}`) |

:::tip
Please consult the official technical documentation for your device or contact device technical support to obtain the correct values for `path`, `method`, `headers`, and `postParams`.
:::

### Example

```javascript
function buildWriteRequest(deviceId, pointId, value) {
    if (!deviceId || !pointId) return {};
    return {
        path: `/device/write/${deviceId}/${pointId}`,
        method: "POST",
        headers: JSON.stringify({
            "Content-Type": "application/json"
        }),
        postParams: JSON.stringify({ value: value })
    };
}
```

## parseWriteResponse 

This method is used to parse the result of the function point write (control) request, in order to determine whether the device control command was executed successfully.


When Aqara Studio sends a write request to the device (such as switching the device on/off, adjusting temperature, etc.) and receives the device's response, it will call this function to judge whether the control was successful based on the returned data.

Please parse the device's returned data in this method and convert the result into a boolean value, so that Aqara Studio can determine whether the control command was executed successfully.

### Parameter Description

It is recommended that this function accepts two string parameters, used to pass in the device ID and the function point write (control) response.

### Return Value

Please ensure that the return value of the function is a boolean, with the following meanings:
- true: Success.
- false: Failure or exception.

### Example

```javascript
function parseWriteResponse(pointId, responseString) {
    if (!responseString) return false;
    let obj;
    try {
        obj = JSON.parse(responseString);
    } catch (e) {
        return false;
    }
    return obj.result === "success";
}
```

## parseReportRequest 

When a device reports data to Aqara Studio (for example, when the device status changes or a sensor uploads data), Aqara Studio will call this function. This function extracts the device ID and the reported data for each function point from the device's report, in order to update the device status.

In this method, please convert the raw data returned by the device into a function point data format recognized by Aqara Studio.

### Prerequisites

Please ensure that the device follows the requirements below for Endpoint, method, and request body when reporting data to Aqara Studio:

- Endpoint: `http://{ip}:8000/report/data`
- Request Method: POST
- Request Body: Customizable by you; just ensure that it can be parsed via the [parseReportRequest](#parsereportrequest) interface to extract the required fields in the return value, i.e., `deviceId` and `points`.

### Parameter Description

This function should accept two string parameters: 
It is recommended that this function accepts two string parameters:
- The first parameter: Pass in the endpoint used by Aqara Studio to receive device data, i.e., `http://{ip}:8000/report/data` as mentioned above.
- The second parameter: The request body of the device's reported data.

Please ensure the parameter names and order are correct so that Aqara Studio can accurately identify and process the device report data for the specified path.

### Return Value

The return value should contain the following three fields:

| Field        | Type                       | Description                                                                                                                      |
|--------------|----------------------------|----------------------------------------------------------------------------------------------------------------------------------|
| deviceId     | String                     | Device ID, used to identify which device the current data belongs to.                                                            |
| points       | List\<Map\<String, String>>| A list of point information, representing multiple data points of the device.<br />Each point is a Map containing: <ul><li>**pointId**: Function point ID </li><li>**value**: Point value.</li></ul> |
| responseMsg  | String                     | Response message, used to inform the device whether Aqara Studio has successfully received this data.                             |

To ensure Aqara Studio can correctly obtain the data of the function points, you need to make sure that the parseReportResponse function returns data in the format shown in the following example:

```json
{
    deviceId: "123456",
    points: [
        { 
            pointId: "temperature", 
            value: "22.5" 
        }
    ],
    responseMsg: ""
}
```

### Example

```javascript
function parseReportRequest(path, requestString) {
    if (!requestString) return { deviceId: "", points: [], responseMsg: "" };
    let obj;
    try {
        obj = JSON.parse(requestString);
    } catch (e) {
        return { deviceId: "", points: [], responseMsg: "" };
    }
    const deviceId = obj.deviceld || obj.deviceId || "";
    const points = [];
    const parans = obj.data && obj.data.parans ? obj.data.parans : {};
    for (const pointId in parans) {
        if (Object.prototype.hasOwnProperty.call(parans, pointId)) {
            const item = parans[pointId];
            points.push({
                pointId: pointId,
                value: String(item.value)
            });
        }
    }
    return { deviceId: deviceId, points: points, responseMsg: "" };
}
```
