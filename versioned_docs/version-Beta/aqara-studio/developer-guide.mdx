# Developer Guide


This document is designed to help developers expand the platform’s capabilities, enabling deep integration and custom development. Whether you want to build personalized applications using the API, or connect devices to mainstream smart home systems, you will find detailed guidance here.

Before starting to use the API v1, you need to [obtain the Studio’s access address and Token](#obtain-required-information). This information is required for request URLs and authentication.

## Obtain Required Information

Before using the API v1, you must first get a Studio access address and Token. These details are used for composing the request endpooint and for authentication.

### Obtain Access Address

You can access Aqara Studio through any of the following three types of addresses:
- IP address (http)
- Host access domain (http)
- Remote access address (https)

The methods to obtain each type are as follows:

- **IP address**: Go to the **Settings** page from the left navigation bar. In the sidebar, choose **Local Settings** to view the **IP address**.
    ![IP Address Acquisition](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Settings-GetIP.png)
- **Host access domain**: See the [Access via LAN Domain Name](./local-domain-name-access.mdx) document for more information.
- **Remote access address**: See the [Remote Access](./remote-access.mdx) document for more details.

### Obtain Token

Follow these steps:

1. In the left sidebar, navigate to the “Developer” page. In the **Data Export API** section, click the **New Token** button.

    ![Click New Token Button](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Developer-NewToken.png)

2. In the popup, define the token name and validity period. Click the **Confirm** button.

    <img src="https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Developer-DefineToken.png" alt="Define Token Name and Validity Period" style={{width: '80%'}} />

    Afterwards, you will see a popup indicating the Token was created successfully and displaying the token.

3. In the **Data Export API** section, you can see the newly added token entry. Click the copy icon to obtain the token.

    ![Copy Token](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Developer-CopyToken.png)

## Call the API

With the complete API, you can programmatically obtain device data, control devices, and subscribe to real-time messages. You may want to:
- Query the device list and their status.
- Remotely control device operations.
- Integrate device data into your own system.
- Develop a customized device management experience.

Please refer to the [API documentation](./api-references/data-export-api-v1.mdx) for detailed API descriptions and usage instructions.
