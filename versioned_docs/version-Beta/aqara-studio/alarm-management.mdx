# Alarm Management

This guide will provide you with a comprehensive understanding of Aqara Studio’s alarm services, including configuration, viewing, and handling of alarms. Through uninterrupted real-time monitoring of all connected devices, the **Alarm Center** presents various alarm information collectively. It allows you to detect and address potential risks promptly and safeguard the security and stability of smart spaces.

## Overview

Aqara Studio offers complete alarm monitoring and management features. It comes with preset default alarm rules for monitoring protocol communication exceptions and device disconnections. Also, it supports user-defined alarm strategies. Therefore, users can detect risks and respond quickly in real time.

- **Multi-dimensional parameter binding**: Device properties (such as temperature, humidity) and business parameters (such as device operation mode) can be linked to trigger alarms dynamically.
- **Typical configuration examples**:
  - When the device temperature continuously exceeds 80°C for 5 minutes, a "High Temperature Alarm" is triggered.
  - When the instantaneous device power drops below 10W, a "Low Power Abnormal Alarm" is triggered.
  - When the device is in "Night Mode" and a motion signal is detected, a "Night Intrusion Alarm" is triggered.

## Check Alarm Information

You can view alarm information on the **Alarm Center > Alarm Information** page. This page displays key details such as the level, source, data, and status of alarm events, making it convenient for users to view and manage alarms in a unified way.

![View Alarm Information](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-CheckAlarms.png)

### Alarm Classes

Alarms can be divided into two classes: **default alarms** and **custom alarms**:

| Dimension | Default Alarm                | Custom Alarm                                                                |
| --------- | ----------------------------------- | ----------------------------------------------------------------------------------------- |
| Meaning | Alarms for protocol communication failures and device offline events. | [User-defined alarms for specific device function points](#配置告警规则). |
| Alarm Level | <ul><li><strong>Critical</strong>: Protocol communication failure</li><li><strong>Warning</strong>: Device offline</li></ul> | Users can customize alarm levels: **Critical**, **Warning**, **Info**. |
| Configured by | Aqara Studio                   | User                                                                                      |
| Configurability | Not modifiable                | Creatable, modifiable, deletable                                                          |
| Trigger Condition | Communication errors, and device offline | Device function point value meets user-defined alarm rule                                 |

:::tip
Default alarms are used to ensure the basic availability of the platform and devices; these rules are maintained by the system, and cannot be modified or deleted by users.
:::

#### Basic Information

The basic information includes the following fields:

| Field                | Description                                                                                                          |
| -------------------- | -------------------------------------------------------------------------------------------------------------------- |
| Alarm level          | The urgency level of the alarm. Options include **Critical**, **Warning**, and **Info**.                             |
| Alarm source         | The source that triggered the alarm.                                                                                  |
| Alarm data           | The detailed description of the alarm. For [custom function point alarms](#alarm-rule-configuration), this shows the alarm name you configured.    |
| Alarm status         | The current status of the alarm. Options include: <ul><li>NORMAL</li><li>OFF_NORMAL</li></ul>                          |
| Alarm time           | The time when the alarm was triggered.                                                                               |
| Return to normal time        | The time when the alarm was cleared and returned to normal.                                                          |
| Alarm ID             | The unique identifier of the alarm.                                                                                  |
| Alarm confirmation status | Whether the alarm has been confirmed or not. Options include: <ul><li>Unconfirmed</li><li>Confirmed</li></ul> |
| Alarm handler        | The user who performed the confirmation operation.                                                                |
| Alarm confirmation time | When the alarm was confirmed.                                                                               |
| Last updated time     | The most recent time the alarm information was updated.                                                              |

#### Alarm History

Alarm history is used to record the complete process of each trigger and recovery for the same alarm. The system aggregates multiple identical and unconfirmed alarm events for easier viewing and tracking.

:::tip
Once a user acknowledges an alarm, if the alarm is triggered again, it will be considered a new alarm and will not be included in the previous alarm history.
:::

Alarm history includes the following fields:

| Field        | Description                                                                                                                                         |
| ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------- |
| Alarm data   | Detailed description of the alarm. For [custom function point alarms](#alarm-rule-configuration), your custom alarm name will be displayed here.                |
| Alarm status | The current status of the alarm: <ul><li>NORMAL</li><li>OFF_NORMAL</li></ul>                                                                          |
| Alarm time   | The specific time when the alarm was triggered.                                                                                                     |
| Return to normal time| The time when the alarm returned to normal.                                                                                                         |

### Alarm Aggregation

The system aggregates all **unconfirmed** alarms of the same rule, and displays them as a single record. By viewing the number in the **Alarm data** field of this record, you can know how many alarms are aggregated together.

## Alarm Rule Configuration

The Alarm Center supports flexible configuration of alarm rules for device function points, enabling real-time monitoring and timely response to abnormal states.

1. Go to **Alarms > Alarm rule** page and click the **Add** button.

    ![Add Alarm Rule](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-AddAlarmRules.png)

2. In the pop-up window, you will see the **Alarm information** section and the **Alarm object** section.

    ![Add Alarm Rule Popup](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-AlarmRuleConfiguration.png)
    
    The **Alarm information** section is used to configure the following parameters:

    | Parameter         | Description                                                                                      |
    |------------------|--------------------------------------------------------------------------------------------------|
    | Alarm name      | The custom name of the alarm.                                                                    |
    | Alarm level      | The urgency level of the alarm. Options include **Critical**, **Warning**, and **Info**.  |
    | Alarm copy | The message shown to the user when the alarm is triggered.   |
    | Return to normal copy | The message shown to the user after recovery.                                                     |
    | Alarm delay       | The system will only send an alarm to the user if the function point stays abnormal for this period. This helps to avoid false alarms caused by short fluctuations.  |
    | Alarm recovery delay | After the function point leaves the alarm state, it must stay normal for this period to be considered recovered. This prevents frequent reminders due to temporary changes. |
    | Effective time | The time when the alarm rule takes effect. |

    In the **Alarm object** section, you can select objects either by **function point** or **function type**.

    - When selecting an object by **function point**：
        - First select the target device, then choose the specific function point under it.
        - Then, configure alarm trigger conditions for this function point:
            - For Boolean types, you can set the alarm to trigger on either true or false;
            - For numeric types, you can flexibly configure upper and lower limits, alarm deadband, and other parameters to achieve refined abnormal monitoring.

        <img src="https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-SelectAPoint.png" alt="Select by function point" />
   
        :::tip
        For numeric function points, the parameters are as follows:

        | Parameter | Description |
        |--|--|
        | Comparison method     | Select the comparison method, such as **out of range**, **greater than**, **less than**, etc.         |
        | Low Limit       | (Only effective when the comparison method is **out of range**) The lowest allowed normal value. When the device reading drops below the low limit, the alarm state changes to OFF_NORMAL (LOW_LIMIT).                                            |
        | High Limit      | (Only effective when the comparison method is **out of range**) The highest allowed normal value. When the device reading exceeds the high limit, the alarm state changes to OFF_NORMAL (HIGH_LIMIT).                                           |
        | Hysteresis        | It is used to prevent frequent state switching due to small fluctuations near the threshold. <ul><li>When the low limit is enabled: Once the reading drops to the low limit and becomes abnormal, the state will only return to normal when the reading rises above the lower limit plus the hysteresis.</li><li>When the high limit is enabled: Once the reading rises to the high limit and becomes abnormal, the state will only return to normal when the reading drops below the high limit minus the hysteresis.</li></ul> |

        The diagram below shows how the alarm state responds as the device reading crosses the low and high limits, as well as when it returns to the recovery points (low limit + hysteresis, and high limit - hysteresis).

        <img src="https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-Hysteresis.png" alt="Alarm status for numeric readings" style={{width: '80%'}} />
        :::
    - When selecting objects by **function type**:
        - First specify the target function type (such as Switch, Temperature, Humidity, etc.), and the system will automatically filter out all function points of that type.
        - Then, you can quickly select all points, select multiple points in bulk, or invert the selection of target function points.
        - Finally, configure the alarm trigger conditions for the selected function points: for Boolean types, you can set the alarm to trigger on true or false; for numeric types, you can flexibly set upper and lower limits, alarm hysteresis, and other parameters for refined abnormal monitoring.

        <img src="https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-SelectPoints.png" alt="Select by function point type" />

3. After configuration is complete, click the **Save** button. The new alarm rule will be added to the list and enabled by default.
4. Once the alarm rule is enabled, whenever the value of a device function point meets the alarm condition, you can view the alarm status, trigger time, and confirm the alarm in [Alarm Information](#check-alarm-information).

## Alarm Template

1. Click **Alarms** in the left navigation bar and select **Alarm Template**. You will see that Aqara Studio provides multiple alarm templates. 

    Click the **Use** button on any template. If there are devices in Aqara Studio that support this rule, you will be able to view and edit the alarm rule in the template drawer.

   ![Enable Alarm Template](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-UseTemplates.png)

2. In the edit drawer, you can configure the alarm rule. After completing the configuration, click the **Save** button.

    ![Save](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-SaveTemplate.png)

3. The newly created alarm rule will be added to the [alarm rule list](#alarm-rule-configuration) and enabled by default.

    ![New alarm rule](https://itad-aqara-docs-1300889962.aqara.com/dev/img/aqara-studio/en/Alarms-NewAlarmByTemplate.png)
