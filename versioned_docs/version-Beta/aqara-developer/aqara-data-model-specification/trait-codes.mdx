## LightingModeSetting

Defines the operating mode of the lighting device.

The LightingModeSetting Trait sets or queries the current operating mode of a lighting device. This mode determines the range of control capabilities available to the device, including on/off control, brightness adjustment, color-temperature adjustment, and color setting. Four modes are currently supported: monochrome (Level), color-temperature (ColorTemp), RGB, and RGBW. The data points and operations exposed by the device vary by mode.

Specifically, in monochrome mode, a light supports only on/off and brightness control; it does not support color-temperature control or color settings, so this mode is suitable for single-color-temperature dimmable lights. In color-temperature mode, the device also supports color-temperature adjustment, enabling smooth transitions from warm white to cool white, but it still does not support RGB or RGBW color settings; this mode is suitable for tunable-white lights. In RGB mode, the device supports full-color RGB settings in addition to on/off, brightness, and color-temperature control (color temperature can be simulated by mixing RGB channels), but does not support an independent white channel. In RGBW mode, an independent white channel is added to RGB mode. It supports four-channel RGBW color settings that can simultaneously provide vivid colors and pure white light for richer lighting scenarios.

Importantly, the operating mode is not an arbitrary software-level setting; it must usually be determined by the light’s physical wiring and actual deployment. For example, a light connected only to warm- and cool-white LEDs must not be set to RGB or RGBW mode, whereas a light with independent RGB+W pins can use RGBW mode. When the mode changes, the device data-model configuration is dynamically updated: data points unsupported in the new mode are automatically hidden, while supported data points are displayed as needed. This ensures that clients correctly identify the device’s current capabilities and avoid issuing invalid commands. Before changing this mode, confirm that the hardware supports the target mode. Make changes cautiously to avoid abnormal device control or a mismatch between the client UI and the actual capabilities.

## ColorMode

The ColorMode trait represents the color space currently used by a lighting device—the color encoding format or coordinate system the device uses to receive and report color data. It supports reading and reporting but not writing. It supports four values: Hue, XY, color temperature, and RGB. Hue describes color using hue, typically with saturation and brightness, and is suitable for intuitive color selection. XY provides an accurate, device-independent color description based on CIE 1931 chromaticity coordinates and is suitable for professional color management. Color temperature describes white light using a color-temperature value (in mireds, `1000000 / color temperature in Kelvin`) to adjust its warmth or coolness. RGB describes colored light using red, green, and blue components, the most common full-color control method. ColorMode and LightingModeSetting are traits at different levels: LightingModeSetting determines the set of functions supported by the device (such as color-temperature or color support), which affects which data-model points are exposed, while ColorMode only determines the encoding format of current color data. They are independent but jointly affect the device's color-control behavior. In particular, when a light is a child device bridged through a gateway, the gateway automatically identifies its supported color spaces and updates its data model. Therefore, when an external system (such as an app or AI controller) controls a light's color, it should determine which color-control commands to send based on the traits and commands in the device data model (for example, whether color-temperature, RGB, or XY control commands are present). The gateway automatically adapts the color space and ultimately controls the child device.
