# Function Codes

## Overview

| functionCode | Description |
| --- | --- |
| [Descriptor](#descriptor) | Describes the composition and capability structure of device endpoints, including endpoint lists, device types, and Server/Client clusters. |
| [BasicInformation](#basicinformation) | Provides basic device information, including the manufacturer, product model, firmware version, serial number, online status, and other core attributes. |
| [EndpointLabel](#endpointlabel) | Manages endpoint visibility and presentation information, such as names, rooms, icons, and tags, for frontend display and organization. |
| [CameraCoordination](#cameracoordination) | Handles complex camera workflows, including WebRTC calls, pan-tilt-zoom control, recognition-data synchronization, and extension events. |
| [GeneralConfiguration](#generalconfiguration) | Provides common device configuration options, such as sensitivity, delays, default parameters, and other device-level settings. |
| [EcosystemInformation](#ecosysteminformation) | Describes the ecosystems to which a device is connected, including binding status for HomeKit, Matter fabrics, and more. |
| [EnergyManagement](#energymanagement) | Manages device battery and energy-consumption information, including battery percentage, power-consumption statistics, and low-battery alerts. |
| [HubCoordination](#hubcoordination) | Serves as the gateway coordination core, managing child-device commissioning, configuration, heartbeats, tokens, and various proxy services. |
| [SceneCoordination](#scenecoordination) | Manages the lifecycle of smart scenes (automation rules), including synchronization, deletion, triggered execution, and consistency validation. |
| [PowerSource](#powersource) | Describes a device’s power source and battery status, including charge level, charging status, battery type, and related information. |
| [WakeupCoordination](#wakeupcoordination) | Obtains a wake-up service address or token for low-power devices that support remote wake-up. |
| [CentralHubCoordination](#centralhubcoordination) | Manages advanced central-hub gateway capabilities, including hub enablement, broker information, and data passthrough channels. |
| [SecurityGuardCoordination](#securityguardcoordination) | Manages home-security arming capabilities, including arming and disarming, alarm acknowledgment, and local/cloud configuration synchronization. |
| [OTACoordination](#otacoordination) | Manages the OTA firmware-upgrade coordination workflow, including task delivery, progress reporting, result feedback, and version queries. |
| [LockCoordination](#lockcoordination) | Handles extended smart-lock functions, including dedicated passthrough channels, encryption/decryption assistance, and extension-event reporting. |
| [DeviceAnomalyMonitoring](#deviceanomalymonitoring) | Continuously monitors device operating parameters to promptly detect and report abnormal or fault conditions. |
| [GeneralDiagnostics](#generaldiagnostics) | Provides general diagnostic information, including uptime, reboot count, network interfaces, and hardware faults. |
| [RingtoneCoordination](#ringtonecoordination) | Manages local gateway ringtone and music files, including downloads, deletion, and consistency validation against the cloud library. |
| [ZigbeeNetworkDiagnostics](#zigbeenetworkdiagnostics) | Provides Zigbee network diagnostics, including channel, PAN, neighbor, and routing information. |
| [MatterCoordination](#mattercoordination) | Centralizes coordination operations related to Matter, including commissioning, binding, data synchronization, and certificate management. |
| [ScreenCoordination](#screencoordination) | Manages smart-scene panel interactions, including configuration delivery, status queries, and bidirectional cloud/panel passthrough. |
| [CommLinkCoordination](#commlinkcoordination) | Monitors the quality of communication links between the gateway and cloud, performs latency checks, and reports results. |
| [IftttCoordination](#iftttcoordination) | Serves as the automation-rule coordination center, managing synchronization, subscriptions, and execution for IFTTT and local automation rules. |
| [NetworkManagement](#networkmanagement) | Serves as the core control module for device network connectivity and access policies, managing remote proxies and access addresses. |
| [Identify](#identify) | Provides device-identification capabilities that help users locate physical devices through indicator flashes, sounds, and similar methods. |
| [LightBoxCoordination](#lightboxcoordination) | Manages configuration delivery for smart light-box devices and controls displayed content and effects. |
| [IRCoordination](#ircoordination) | Manages gateway infrared learning and transmission, including code-library synchronization, virtual remote creation, and button control. |
| [NetworkCommissioning](#networkcommissioning) | Provides network commissioning capabilities, including scanning, connecting to, and configuring Wi-Fi, Thread, and other networks. |
| [ConfigurationCoordination](#configurationcoordination) | Manages common-function configuration, resource subscriptions, and location-information synchronization through multicast or batch operations. |
| [CloudStorageCoordination](#cloudstoragecoordination) | Manages cloud recording and storage service resources, obtaining storage quotas, upload credentials, and related information. |
| [StudioCoordination](#studiocoordination) | Manages integration between gateways and the Aqara Studio development environment, including account authentication, project backups, and device synchronization. |
| [Temperature](#temperature) | Measures and reports ambient temperature, providing attributes such as the current temperature and valid measurement range. |
| [TemperatureControl](#temperaturecontrol) | Provides temperature-setting and adjustment capabilities for target-temperature control on thermostatic devices. |
| [PackageRecognition](#packagerecognition) | Automatically recognizes packages in a video frame and can send reminders and record events when packages are delivered or removed. |
| [FaceRecognition](#facerecognition) | Captures facial features, compares them with a local library, and reports facial-recognition results and match information. |
| [HumanPresence](#humanpresence) | Detects human presence in a space and supports advanced sensing capabilities such as people counting, trajectories, and identity association. |
| [HumanRecognition](#humanrecognition) | Uses vision and other algorithms to detect human figures in real time, distinguish people from other objects, and report events. |
| [AirQuality](#airquality) | Monitors and reports air-quality indicators, such as AQI or an overall air-quality level. |
| [AttitudeDetection](#attitudedetection) | Detects changes in device orientation or tilt angle and reports orientation anomalies or angle information. |
| [Lifter](#lifter) | Provides travel control for lifting devices, supporting target-position settings, stopping, and status queries. |
| [Binding](#binding) | Represents a persistent binding relationship between an endpoint and one or more other local or remote endpoints. |
| [PetFeeder](#petfeeder) | Provides pet-feeding controls, including portion counts, feeding schedules, and remaining-food status. |
| [Button](#button) | Describes event reporting for button input devices, including recognition of single clicks, long presses, and multiple clicks. |
| [Output](#output) | Provides switched-output control, supporting reading and controlling the on/off state of a load. |
| [IRControl](#ircontrol) | Provides infrared-transmission control for conventional infrared appliances such as air conditioners and televisions. |
| [HeaterCooler](#heatercooler) | Provides cooling and heating control, including HVAC parameters such as target temperature, operating mode, and fan speed. |
| [Vibration](#vibration) | Detects vibration or motion-state changes and reports events such as movement, tilting, and dropping. |
| [Cube](#cube) | Describes gesture recognition for cube controllers, including flips, rotations, shakes, and face recognition. |
| [Knob](#knob) | Describes rotation-angle, press, and combined-action events for knob input devices. |
| [BinaryOutputObject](#binaryoutputobject) | Drives binary actuator outputs, such as relays, contactors, solenoid valves, or sirens. |
| [AnalogInputObject](#analoginputobject) | Acquires continuously varying analog inputs in read-only mode, such as voltage and simulated temperature signals. |
| [MediaPlayback](#mediaplayback) | Provides media-playback control, including play, pause, track changes, volume, and playback-state management. |
| [MediaInput](#mediainput) | Describes media input-source management and switching, supporting selection of the current playback source and input list. |
| [FlameDetection](#flamedetection) | Detects visible flames through image or sensor algorithms and immediately reports fire events for early warning. |
| [OperationalState](#operationalstate) | Describes the device’s current operational state, such as idle, running, paused, or faulted. |
| [RefrigeratorState](#refrigeratorstate) | Describes refrigerator operational states, including idle, cooling, defrosting, and faults. |
| [Mode](#mode) | Provides a device feature-mode interface that supports switching among multiple predefined modes. |
| [BridgedConfiguration](#bridgedconfiguration) | Describes configuration information for bridged child devices in Matter/Zigbee bridging scenarios. |
| [MultiStateInputObject](#multistateinputobject) | Acquires multi-state discrete inputs that represent one of N states. |
| [MARSConfiguration](#marsconfiguration) | Provides smart-load (MARS) configuration for load identification and intelligent power-distribution management. |
| [ScreenUserInterface](#screenuserinterface) | Manages screen-side user-interface configuration, including page layouts, cards, themes, and displayed data. |
| [CustomizedConfiguration](#customizedconfiguration) | Provides read and write access to manufacturer- or user-defined extension configuration items. |
| [HEPAFilterMonitoring](#hepafiltermonitoring) | Monitors HEPA filter use, remaining life, and replacement reminders. |
| [ActivatedCarbonFilterMonitoring](#activatedcarbonfiltermonitoring) | Monitors activated-carbon filter use, remaining life, and replacement reminders. |
| [FanControl](#fancontrol) | Provides fan-speed and mode control, supporting speed-level adjustment and operating-state reading. |
| [HeartMonitoring](#heartmonitoring) | Uses sensors to continuously or periodically measure user heart rate and record and analyze heart-health data. |
| [Pressure](#pressure) | Measures and reports atmospheric or system pressure values. |
| [MultiStateOutputObject](#multistateoutputobject) | Outputs multi-state discrete control commands, such as HVAC mode or lighting-scene changes. |
| [Contact](#contact) | Detects binary contact or door-sensor states and reports open/closed states and optional angle information. |
| [Doorbell](#doorbell) | Provides doorbell capabilities, including call-event reporting and integration with indoor chimes or relays. |
| [WaterFreezeDetector](#waterfreezedetector) | Detects freezing risk in pipes or the environment and reports water-freezing events when thresholds are reached. |
| [Speaker](#speaker) | Describes device speaker output capabilities, including audio-output attributes such as volume control and mute. |
| [DoorLock](#doorlock) | Provides core smart-lock capabilities, including lock status, user/credential management, unlocking/locking, and alarm events. |
| [CO2](#co2) | Measures and reports ambient carbon-dioxide concentration. |
| [VacuumOperationalState](#vacuumoperationalstate) | Describes the current operational state of a robot vacuum, such as cleaning, returning to charge, or charging. |
| [ValveControl](#valvecontrol) | Provides automatic or manual valve control for precise adjustment of water or gas flow. |
| [ScreenDisplay](#screendisplay) | Manages screen-display hardware attributes, such as resolution, brightness, backlight, and display state. |
| [Illuminance](#illuminance) | Measures and reports ambient illuminance (lux). |
| [Camera](#camera) | Manages core camera capabilities, including device status, mode switching, image encryption, and cloud-related functions. |
| [Gas](#gas) | Detects combustible or harmful gas leaks and reports alarm events. |
| [GestureRecognition](#gesturerecognition) | Analyzes hand gestures in real time and converts predefined gestures into control commands. |
| [BreathingMonitoring](#breathingmonitoring) | Uses sensors to continuously or periodically measure user breathing rate and patterns for health monitoring. |
| [CharacterStringValueObject](#characterstringvalueobject) | Defines data objects represented as strings and their externally visible characteristics. |
| [Sweeper](#sweeper) | Provides basic robot-vacuum controls, including starting cleaning, pausing, returning to charge, and status feedback. |
| [EVSE](#evse) | Provides electric vehicle supply equipment (EVSE) control, including starting and stopping charging, current limits, and charging status. |
| [CO](#co) | Provides carbon-monoxide concentration monitoring and alarm capabilities to detect ambient carbon-monoxide leaks and issue timely alerts. |
| [Lighting](#lighting) | This Function provides the ability to activate and control dynamic lighting effects.The server provides a list of effects driven by different sources, from which clients can select and trigger effects. Dynamic lighting effects can be used both with luminaires that support the Color Control Function and with luminaires that support only the Level Control Function (for example, dimmable lights). The specific implementation of effects is manufacturer-defined.Even when devices from different manufacturers use the same EffectID, they are not guaranteed to represent the same effect; a manufacturer may reuse an EffectID across devices to represent a consistent implementation. |
| [LevelControl](#levelcontrol) | Provides brightness-level adjustment, supporting setting and reading the current brightness, step dimming, and dimming control coupled with or decoupled from on/off control. |
| [ColorControl](#colorcontrol) | Provides color control, supporting adjustment of lighting color through HSV, XY, and color-temperature modes, including combined delivery with brightness commands. This Function provides an interface for adjusting lighting color.Color definitions follow the CIE 1931 color space, with x and y chromaticity coordinates as the primary control method (corresponding to ColorXY / CurrentX / CurrentY). Additionally, color can be controlled through ColorTemperature (color temperature) or Hue + Saturation; Hue and Saturation are recommended to be interpreted using the HSV (HSB) model. RGB/W primary-point capabilities can extend the range of color expression. This Function does not control brightness; brightness is provided by the LevelControl Function. When used together, the brightness value in LevelControl should be interpreted as a proportion of the maximum achievable light intensity for the current color. |
| [PanTilt](#pantilt) | Provides pan/tilt/zoom control, including presets, calibration, and relative or absolute movement. |
| [WindowCovering](#windowcovering) | This Function provides open/close control, tilt control, target-position management, and operating-state feedback for window coverings such as curtains, roller shades, and blinds. In addition to standard open/close and tilt control, this Function includes Aqara extensions: scene management (starting, reading, previewing, adding, deleting, and editing built-in/custom scenes), manual calibration, remote pairing, motor-mapping exchange, and enhanced-mode status. Clients should use Traits such as TravelConfigured, MotorControllability, MotorOperationStatus, and EnhanceMode to determine the device’s current controllability and operation phase, avoiding invalid commands when requirements are not met. |
| [Flow](#flow) | Measures and reports the flow rate of liquids or gases. |
| [OccupancySensing](#occupancysensing) | Detects whether people are present in a space and reports occupied/vacant status and parameters such as hold time. |
| [RainSensor](#rainsensor) | Detects rainfall status and reports whether it is raining and related precipitation information. |
| [Smoke](#smoke) | Detects smoke and reports alarm events, supporting related configuration such as sensitivity, self-test, and mute. |
| [FreshAirControl](#freshaircontrol) | Provides fresh-air system control, supporting outdoor-air intake and adjustment of airflow, mode, and other settings. |
| [AnalogOutputObject](#analogoutputobject) | Provides writable continuous analog output control signals. |
| [Location](#location) | Provides location-information reporting and query capabilities for devices or targets. |
| [PetRecognition](#petrecognition) | Recognizes pets such as cats and dogs in a video frame and reports pet appearances or related recognition events. |
| [ParticulateMatter](#particulatematter) | Monitors and reports particulate-matter concentrations, such as PM2.5 and PM10. |
| [OnOffSensor](#onoffsensor) | Detects binary switch-input states and reports discrete states such as on/off or triggered/not triggered. |
| [SoundDetection](#sounddetection) | Captures ambient sounds through a microphone, recognizes specific sound types (such as breaking glass or a crying baby), and raises alerts. |
| [Ringtone](#ringtone) | Provides ringtone and music-playback control, supporting track selection, volume/duration adjustment, and playback-state monitoring. |
| [BridgeConfiguration](#bridgeconfiguration) | Manages gateway bridging configuration and describes child-device access, bridging capabilities, and network-topology information. |
| [MatterBridgeCommissioning](#matterbridgecommissioning) | Manages the ability of a Matter Bridge to share bridged devices with third-party Matter ecosystems. |
| [GarageDoorOpener](#garagedooropener) | Provides garage-door control and status feedback, including open, close, stop, and position status. |
| [RelativeHumidity](#relativehumidity) | Measures and reports ambient relative humidity, providing attributes such as the current humidity and valid measurement range. |
| [Sleep](#sleep) | Monitors sleep-related states and reports metrics such as sleep stage, duration, and quality. |
| [BinaryInputObject](#binaryinputobject) | Acquires binary state inputs, such as door sensors and switches, represented as true/false or active/inactive values. |
| [WaterLeak](#waterleak) | Detects water-leak events and promptly reports an alarm when a leak is detected. |
| [VehicleDetection](#vehicledetection) | Detects vehicles appearing in a monitored area and reports vehicle-presence events for security or automation use. |

## ColorControl

Provides color control, supporting adjustment of lighting color through HSV, XY, and color-temperature modes, including combined delivery with brightness commands.

This Function provides an interface for adjusting lighting color.Color definitions follow the CIE 1931 color space, with x and y chromaticity coordinates as the primary control method (corresponding to ColorXY / CurrentX / CurrentY).
Additionally, color can be controlled through ColorTemperature (color temperature) or Hue + Saturation; Hue and Saturation are recommended to be interpreted using the HSV (HSB) model. RGB/W primary-point capabilities can extend the range of color expression.
This Function does not control brightness; brightness is provided by the LevelControl Function. When used together, the brightness value in LevelControl should be interpreted as a proportion of the maximum achievable light intensity for the current color.
