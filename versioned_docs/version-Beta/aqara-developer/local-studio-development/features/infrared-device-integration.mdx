# Infrared Device Integration

本文描述如何通过 Studio 本地 API 将红外设备接入 Aqara Studio，从而通过网关下发红外指令控制红外设备。

## 时序图

## 流程

### 1 创建红外设备

通过 Studio 本地 API [`POST` /open/api/v1/virtual-devices](./../../data-export-api/add-virtual-device.api.mdx) 接口，创建一个红外设备，获取该红外设备 ID。

### 2 查询设备模型定义

通过 Studio 本地 API [`GET` /open/api/v1/devices/definitions](./../../data-export-api/get-device-definitions.api.mdx) 接口，传入红外设备 ID 及目标网关 ID，查询设备模型定义，以便获取后续操作所需的 **endpointId**、**functionCode** **commandCode**。

常用 commandCode 说明如下： 

| 类别       | 功能            | commandCode（括号内为 commandId）                  |
|------------|-----------------|--------------------------------------------|
| 红外设备   | 获取设备类型     | `APIGetIrDeviceCategoriesForApp(40263)`    |
| 红外设备   | 获取品牌         | `APIGetBrandsByCategoryForApp(40267)`      |
| 红外设备   | 获取匹配树       | `APIGetMatchTreeForApp(40268)`             |
| 红外设备   | 试码             | `APIClickIrKeyForApp(40273)`               |
| 红外设备   | 保存匹配         | `APISaveMatchedRemoteInfoForApp(40292)`    |
| 红外设备   | 创建设备         | `APICreateIrVirtualDeviceForApp(40281)`    |
| 网关       | 下载码库         | `SetIRDeviceInfo(40288)`                   |
| 网关       | 等待状态         | `IRCodeMatchStatus(20591)`                 |
| 网关       | 日常控制         | `SendIrKey(40305)` <br />**注意**：此 Command 仅在匹配完成后出现，因此，您需要在匹配完成后再次查询设备模型定义，才能获取到该 Command。                         |
| 网关       | 读取学习结果     | `IRLearnResult(20618)`                     |

接口：`SendIrKey(40305)`

# 红外码匹配接口文档（含流程、输入输出与实际示例）

---

## 1. 匹配红外流程

### 1.1 红外码库匹配流程

1. 获取设备类型：`APIGetIrDeviceCategoriesForApp(40263)`  
2. 获取品牌：`APIGetBrandsByCategoryForApp(40267)`  
3. 获取匹配树：`APIGetMatchTreeForApp(40268)`  
4. 试码：`APIClickIrKeyForApp(40273)`  
5. 保存匹配：`APISaveMatchedRemoteInfoForApp(40292)`  
6. 下载码库：`SetIRDeviceInfo(40288)`  
7. 等待状态：`IRCodeMatchStatus(20591)`  
8. 创建设备：`APICreateIrVirtualDeviceForApp(40281)`  
9. 日常控制：`SendIrKey(40305)`

### 1.2 学习码流程

1. 开启学习：`APIStartIrLearningForApp(40280)`  
2. 读取学习结果：`IRLearnResult(20618)`  
3. 保存学习按键：`APIAddIrKeyForApp(40275)`  
4. 创建设备时自动下发学习码

## 4. APIGetMatchTreeForApp

- CommandId: `40268`
- 代码对应：`TreeCategoryCommandPoint`
- 文件路径：
  - CategoryTree：`...\matchTree\category\<CategoryId>\brand\<BrandId>`
  - SpsCityTree：`...\matchTree\city\<CityId>\sp\<SpId>`
  - IptvTree：`...\matchTree\iptv\<BrandId>`

### 输入参数

- `Type`：integer，必填
  - `1` CategoryTree
  - `2` SpsCityTree
  - `3` IptvTree
- `CategoryId`：integer，Type=1 时必填
- `BrandId`：integer，Type=1/3 时必填
- `CityId`：integer，Type=2 时必填
- `SpId`：integer，Type=2 时必填

### 输出参数

- `result`：`string`（JSON 字符串，匹配树）

### 输入示例（CategoryTree）

```json
{
  "Type": 1,
  "CategoryId": 2,
  "BrandId": 1007
}
```
